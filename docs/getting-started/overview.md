# LedgerLeap 概要

## 概要

LedgerLeap は、組織内の情報を**台帳（レコード）** として一元管理し、検索・共有・承認・追跡を効率化する Web ベースの情報管理プラットフォームです。散在しがちな業務記録やノウハウを構造化して蓄積し、必要な人が必要な情報に素早くアクセスできる環境を提供します。

## LedgerLeap で解決できること

- **情報の散在**: メール、共有フォルダ、紙の記録など複数の場所に分散している業務情報を、1 つのシステムに統合します。
- **情報発見の困難さ**: 台帳データと添付ファイルの両方を対象にした全文検索により、目的の情報を素早く見つけられます。
- **アクセス制御の複雑さ**: フォルダ階層とロールベースの権限により、情報の閲覧・編集・承認を細かくコントロールします。
- **変更追跡と監査**: すべての操作が活動履歴として記録され、変更の経緯をいつでも確認できます。

## 基本概念

### テナント

テナントは、システムを利用する組織やプロジェクトの単位です。LedgerLeap はマルチテナント方式を採用しており、1 つのシステム上で複数の組織がデータを論理的に分離して運用できます。ユーザーは複数のテナントに所属し、テナントを切り替えながら業務を行えます。

### フォルダ

フォルダは、台帳（レコード）を整理するための階層的な入れ物です。フォルダごとにアクセス権限を設定でき、ユーザーは権限のあるフォルダとその配下の台帳のみを閲覧・操作できます。

### 台帳定義（LedgerDefine）

台帳定義は、どのような項目（カラム）を持つ台帳かを指定するテンプレートです。テキスト、数値、日付、自動採番、Markdown、添付ファイルなど、用途に合わせて自由にカラムを設定できます。

### 台帳レコード（Ledger）

台帳定義に基づいて作成される個々のデータレコードです。日報、案件管理、在庫記録など、あらゆる業務データをこのレコードとして保存します。

### ワークフロー

台帳レコードに対して、点検→承認といった多段階の承認プロセスを設定できます。ワークフローが有効な台帳では、登録・更新時に指定された承認者の判断を経てから確定します。

## ユーザー種別と役割

| ユーザー種別 | 主な業務 |
|---|---|
| 一般ユーザー | 台帳の閲覧・登録・編集、添付ファイルの確認、検索 |
| 点検者・承認者 | ワークフローにおける点検・承認タスクの実行 |
| 管理者（Filament） | ユーザー管理、フォルダ・権限設定、テナント設定、タグ・類義語管理 |

## 最初の一歩

1. 管理者から**テナントとフォルダへのアクセス権**を付与してもらいます。
2. **My Portal** にログインし、自分の担当フォルダや通知、承認待ちタスクを確認します。
3. フォルダツリーから目的のフォルダを選び、台帳一覧を開きます。
4. 台帳を開いて内容を確認したり、新しくレコードを作成したりします。

詳細は [My Portal とナビゲーション](portal-and-navigation.md) を参照してください。
