## 🐛 問題の詳細

### 改善前の問題

#### タグの問題点

1. **営業日報に23個のタグ**
   - 「新規案件」「既存顧客」「フォローアップ」「成約済み」「見送り」
   - 「開発」「保守」「障害対応」「改善提案」「テスト」
   - 「承認済み」「確認待ち」「差戻」
   - 「2025年度営業計画」「新製品展開」「顧客管理」「品質改善」「DX推進」
   - 「緊急」「日次」

2. **他の台帳定義には少数のタグ**
   - 経費申請: 「申請中」「月次」（2個）
   - 設備点検表: 「報告」「定例」（2個）
   - 週報: 「週次」（1個）

3. **横断検索ができない**
   - 各台帳定義に固有のタグが付いている
   - 複数の台帳定義をまたいで検索できない
   - タグ名が具体的すぎて汎用性がない

### タグの本来の目的

タグは**フォルダとは別の切り口で、複数の台帳定義を横断的にグループ化する**ための機能：

| 分類方法 | 用途 | 例 |
|---------|------|-----|
| **フォルダ** | 組織構造・部門別の分類 | 営業部/日報、技術部/週報 |
| **タグ** | 属性・用途による横断的な分類 | 「重要」「緊急」「月次報告」 |

## ✅ 改善内容

### タグ設計の原則

1. **汎用性**: 複数の台帳定義で使える名前
2. **横断性**: 部門やフォルダを越えて検索できる
3. **適度な抽象度**: 具体的すぎず、抽象的すぎない
4. **体系性**: カテゴリごとに整理された命名

### 新しいタグ体系

#### カテゴリ別タグ一覧

| カテゴリ | タグ名 | 用途 |
|---------|--------|------|
| **頻度** | 日次報告、週次報告、月次精算、月次点検 | 報告・処理の頻度 |
| **業務種別** | 営業活動、開発活動、顧客対応、経理処理、設備管理 | 業務の種類 |
| **ステータス** | 要承認、進捗管理 | 処理状態 |
| **用途** | 安全確認 | 特定の目的 |

## 🎯 タグ設計のベストプラクティス

### 良いタグの例

✅ **頻度系**: 日次、週次、月次、年次  
✅ **業務系**: 営業活動、開発活動、経理処理  
✅ **状態系**: 要承認、進行中、完了  
✅ **重要度系**: 重要、緊急、通常  
✅ **用途系**: 報告、申請、記録、点検

### 避けるべきタグの例

❌ **年度限定**: 2025年度営業計画、2024年プロジェクト  
❌ **具体的すぎる**: 株式会社A商事、田中太郎  
❌ **レコード固有**: 新規案件、見送り、成約済み  
❌ **ステータスと重複**: 承認済み、確認待ち、差戻（ワークフローステータスで管理すべき）

## 🚀 実装方法

### Seederでの実装

```php
// DemoPhase1ExtensionSeeder.php
private function createTags(): void
{
    $tagAssignments = [
        '[DEMO] 営業日報' => [
            '日次報告',     // 頻度
            '営業活動',     // 業務種別
            '顧客対応',     // 業務種別
        ],
        
        '[DEMO] 経費申請' => [
            '月次精算',     // 頻度
            '経理処理',     // 業務種別
            '要承認',       // ステータス
        ],
        
        '[DEMO] 設備点検表' => [
            '月次点検',     // 頻度
            '設備管理',     // 業務種別
            '安全確認',     // 用途
        ],
        
        '[DEMO] 週報' => [
            '週次報告',     // 頻度
            '進捗管理',     // 用途
            '開発活動',     // 業務種別
        ],
    ];

    foreach ($tagAssignments as $defineTitle => $tagNames) {
        $define = LedgerDefine::where('title', $defineTitle)->first();
        foreach ($tagNames as $tagName) {
            Tag::firstOrCreate(
                ['name' => $tagName, 'ledger_define_id' => $define->id],
                ['creator_id' => $user->id, 'folder_id' => $define->folder_id]
            );
        }
    }
}
```
