# LedgerLeapにおけるAIアシスタントと検索の哲学

## はじめに

LedgerLeapのAIアシスタントは、単なるチャットボットではありません。あなたの業務を深く理解し、必要な情報を迅速かつ正確に提供する、信頼できるパートナーです。

このドキュメントでは、AIアシスタントがどのように情報を探し、どのような考え方であなたをサポートするのか、その裏側にある2つの強力な検索アプローチと、それぞれの使い分けについて解説します。これを理解することで、あなたはAIアシスタントをより効果的に活用できるようになります。

## 2つの検索アプローチ：「事実の発見」と「知識の発見」

AIアシスタントは、あなたの質問の意図を汲み取り、大きく2つのアプローチを使い分けています。

### 1. LedgerLeap MCP検索：信頼できる「事実の発見」

これは、LedgerLeapの根幹をなす、**信頼性を最優先**する検索アプローチです。

- **目的:** 「承認済みの契約書はどこか？」「Aプロジェクトの担当者は誰か？」といった、**正解が一つに定まる「事実（ファクト）」**を、迅速かつ正確に探し出します。
- **仕組み:** 台帳のメタデータ（ステータス、担当者、日付など）と、添付ファイルの全文テキストを組み合わせた、厳密な検索を行います。ワークフローの状態や権限設定も完全に反映されます。
- **得意なこと:**
    - 条件（日付、担当者、ステータスなど）に基づいた正確な絞り込み。
    - 業務プロセスに直結したレコード（証跡）の特定。
- **アシスタントの役割:** 信頼できる**「データアナリスト」**や**「ファイリング担当者」**。

### 2. RAG (Retrieval-Augmented Generation)：探索的な「知識の発見」

これは、複数の情報源から関連する知識を抽出し、新しい洞察を生み出すための、**探索的なアプローチ**です。

- **目的:** 「過去の提案書から、顧客に響いたセールストークのパターンを教えて」「複数の技術仕様書を比較して、それぞれの強みを要約して」といった、**答えが一つに定まらない問い**に対して、示唆に富む回答を生成します。
- **仕組み:** 文章の「意味」を理解し、表現が異なっていても文脈が近い情報の断片（チャンク）を複数のドキュメントから横断的に集め、それを元にLLMが回答を合成します。
- **得意なこと:**
    - 複数の長文ドキュメントの要約・比較。
    - アイデア出しやブレインストーミングの壁打ち。
- **アシスタントの役割:** 発想豊かな**「リサーチアシスタント」**。

## 検索技術の裏側：なぜ違いが生まれるのか？

「似ているものを探す」という点では同じでも、その「物差し」が根本的に異なります。

| 比較軸 | LedgerLeap検索 (Mroonga/TF-IDF) | RAG (ベクトル検索) |
| :--- | :--- | :--- |
| **物差し** | **単語の出現頻度** | **文章の意味・文脈** |
| **検索の性質** | キーワード検索の延長 | 意図解釈・概念検索 |
| **アナロジー** | **索引（インデックス）**で探す百科事典 | **文脈を理解する**優秀な司書 |

#### 類語検索を組み合わせても越えられない壁

LedgerLeapは、キーワードの揺れを吸収するために類語検索も活用します。これはキーワード検索を大幅に賢くしますが、それでも「意味」そのものを捉えるセマンティック検索（ベクトル検索）とは原理的な違いがあります。

- **類語辞書の限界:** 辞書にない新しい言葉や、文章全体の構造から生まれる意味（例：「猫が魚を食べる」と「魚が猫を食べる」の違い）を理解するのは困難です。
- **セマンティック検索の強み:** 文章構造や文脈を理解するため、辞書に頼らずとも柔軟に意味の近さを判断し、人間が思いもよらない関連性（セレンディピティ）を発見することがあります。

## 実践的な使い分け：あなたの目的はどちらですか？

| あなたの目的 | 推奨アプローチ | なぜなら |
| :--- | :--- | :--- |
| **承認済みの契約書**など、特定のレコードを探したい | **LedgerLeap MCP検索** | システムが「承認済」という事実を保証しており、最も信頼性が高いから。 |
| **先月の自分の活動**を正確に振り返りたい | **LedgerLeap MCP検索** | 「作成者: 自分」「期間: 先月」で検索すれば、ノイズなく正確な活動履歴が得られるから。 |
| **過去の類似案件**から、**新しい提案のアイデア**を得たい | **RAG** | 複数の提案書を横断的に読み解き、共通のパターンや新しい切り口を「合成」するのが得意だから。 |
| **複数の製品仕様書**の**要点を比較**したい | **RAG** | 長文ドキュメントから関連箇所だけを効率的に抽出し、比較・要約レポートを生成できるから。 |

## まとめ：LedgerLeapが目指す未来

LedgerLeapのAIアシスタントは、これら2つのアプローチを適切に使い分けることで、**「確実な事実の提示」** と **「創造的な知識の発見」** の両立を目指します。

まずは、業務システムの根幹である**信頼性**を担保するためにLedgerLeap MCP検索を主軸としつつ、ユーザーの探索的・創造的な活動を支援するためにRAGのアプローチで補完します。

これにより、あなたは日々の定型業務を効率化し、さらにデータに基づいた新しい価値創造に集中することができるようになります。
