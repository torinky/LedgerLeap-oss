# UIマイグレーション・プレイブック

LedgerLeapプロジェクトの各画面を、daisyUI + Tailwind CSS v4の標準デザインルールに移行するための標準手順です。

関連資料:
- [Design Instructions](/.github/instructions/design.instructions.md)
- [Agent Routing](/AGENTS.md)

---

## 1. 目的

画面ごとにバラついていたハードコードのCSSや旧来のHTML構造を廃止し、統一されたデザイン方針（`design.instructions.md`）に沿った一貫性のあるUIへ移行する。
特に **Mary UI コンポーネントを優先** し、それをプロジェクトのテーマ（`corporate` / `coffee`）に適合させることを目的とする。

---

## 2. 対象

すべてのBladeビュー、及びLivewireコンポーネントにおけるUI出力箇所。

---

## 3. 標準フロー

画面のマイグレーションに着手する際は、毎回以下の順番で作業を行う。

### Step 1 — 既存構造の分析
対象となるBlade/Livewireビューファイルを確認し、以下の要素を洗い出す。
- raw HTMLで作られたUIパーツ（ボタン、カード、インプット等）
- ハードコードされた色指定（Hex値）や独自の間隔調整（`mt-[15px]` や `padding: 10px;` など）
- 古い仕様（手書きのインラインスタイルなど）

### Step 2 — Mary UI コンポーネントへの置換
見つけた raw HTML を、可能な限り Mary UI コンポーネントに置き換える。
- 例: `<div class="card">...</div>` → `<x-card>...</x-card>`
- 例: `<button class="...">` → `<x-button ... />`
- 例: `<input type="text" ...>` → `<x-input ... />`

### Step 3 — daisyUI/Tailwind セマンティックの適用
`design.instructions.md` のルールに従い、色と余白を置き換える。
- 独自の色指定を、daisyUI のセマンティックカラー（`primary`, `base-100`, `text-base-content`, `info`, `warning`, `error` 等）に完全に置き換える。
- サイズや余白指定は Tailwind の標準スケール（`p-4`, `m-2`, `gap-4` 等）へ変換する。

### Step 4 — ダーク/ライトテーマへの適合確認
- プロジェクトで定義されたLight(`corporate`) と Dark(`coffee`) の両テーマで破綻なく表示できるか確認する。
- 視認性やコントラストの問題があれば、Mary UI のコンポーネント属性か、Tailwindのセマンティッククラスで補正する。

### Step 5 — デバイス・レスポンシブ対応の確認（ノートPCとタブレット）
- 現場（タブレット）と事務所（ノートPC）での利用を想定し、両方のブレークポイントで検証する。
- **タブレット (`md:` 以下)**: 要素が窮屈になっていないか、操作ボタンが指でタップしやすいサイズを維持しているか確認する。ホバーアクションに依存していないか確認する。
- **ノートPC (`lg:` 以上)**: 広い画面幅を活かして情報（テーブルやカード）が一覧性高く表示されているか確認する。

### Step 6 — 不用なCSSのクリーンアップ（任意）
- この画面のマイグレーションにより、完全に不要となった `app.scss` 等のカスタムCSSクラスがあれば調査し、他画面への影響がないことを確認した上で削除・整理する。

---

## 4. 完了条件

- [ ] 対象画面から色やピクセルのハードコードが排除されている
- [ ] 適用可能な箇所で Mary UI コンポーネントが使用されている
- [ ] `corporate`, `coffee` の両テーマでテキストやボタンの視認性が落ちていない
- [ ] ノートPCとタブレットの両方の画面幅（利用シーン）において表示崩れがなく、十分な操作性が確保されている
- [ ] AIが後から見て再加工できる状態（Tailwind準拠のクリーンな構造）になっている

---

## 5. 推奨運用

- 特定の画面をリファクタリングする際は、AIエージェントに「`ui-migration-playbook.md` というプレイブックに従ってこの画面を移行してください」と指示する。
- `design.instructions.md` に記載のない例外的なスタイルが必要になった場合は、このプレイブックを適用する過程でルール化の可能性を検証し、汎用性があれば `design.instructions.md` に追記する。
