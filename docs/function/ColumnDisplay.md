# 台帳の表示・入力UIの改善

## 1. 概要

LedgerLeapでは、一つの台帳に多数の項目（カラム）が含まれる場合でも、ユーザーが情報を効率的に閲覧・入力できるよう、以下の2つのUI改善機能を導入しています。

1.  **カラムのグループ化:** 関連する入力項目をグループにまとめ、折りたたみ可能なアコーディオン形式で表示します。
2.  **表示レベル制御:** 「概要」「詳細」のように、表示する情報の粒度をユーザーが動的に切り替えられるようにします。

これらの機能は、台帳の一覧画面、詳細画面、作成・編集フォームの全てに適用され、一貫したユーザー体験を提供します。

## 2. 機能詳細

### 2.1. 管理者による設定

これらの表示制御は、管理者が台帳定義画面で各カラムに対して以下の2つの属性を設定することで実現されます。

-   **グループ名:**
    *   カラムが属するグループの名前を文字列で指定します。同じグループ名を持つカラムは、UI上で一つのグループとしてまとめられます。
    *   グループ名が設定されていないカラムは、「その他」のようなデフォルトグループに分類されます。
-   **表示レベル:**
    *   カラムの重要度を `1` (概要), `2` (詳細), `3` (全項目) の3段階で設定します。数値が小さいほど重要度が高くなります。

### 2.2. カラムのグループ化（入力・詳細画面）

台帳の作成・編集フォーム、および詳細表示画面では、カラムが「グループ名」に基づいて、折りたたみ可能なアコーディオンパネルで表示されます。

-   **直感的な情報整理:** 業務の流れに沿って「基本情報」「測定値」のように項目がグループ化されるため、ユーザーはフォームの全体像を把握しやすくなります。
-   **入力効率の向上:** 普段あまり使わないグループを閉じておくことで、不要なスクロールを削減し、目的の入力項目に素早くアクセスできます。
-   **必須項目の可視化:** 入力必須の項目が含まれるグループには、その旨を示すインジケーターが表示され、入力漏れを防ぎます。

### 2.3. 表示レベル制御（一覧・入力・詳細画面）

台帳の一覧画面、入力フォーム、詳細画面には、表示レベルを切り替えるためのボタン（例: 「概要」「詳細」「全項目」）が設置されています。

-   **段階的な情報表示:**
    *   **概要 (レベル1):** 「プロジェクト名」「ステータス」など、最も重要な項目のみが表示されます。一覧画面のデフォルト表示であり、状況の素早い把握に適しています。
    *   **詳細 (レベル2):** 概要レベルの項目に加え、より詳細な情報が表示されます。
    *   **全項目 (レベル3):** 全ての項目が表示されます。
-   **視認性の向上:** ユーザーは、目的に応じて表示する情報の粒度を動的に変更できます。これにより、特に多数のカラムを持つ台帳一覧画面での水平スクロールの手間が軽減され、複数レコードの比較・分析が容易になります。
-   **状態の維持:** 選択した表示レベルはURLのクエリ文字列に反映されるため、ページをリロードしたり、URLを共有したりしても表示状態が維持されます。
