# テストコード

## 概要

LedgerLeap には、様々な機能を検証するためのテストコードが実装されています。これらのテストコードは、新しい機能を追加したり、既存の機能を修正したりする際に、システムの安定性を確認するために役立ちます。

## 機能詳細

### テストコードの種類

* **Unit テスト**: 個々のコンポーネントやメソッドが正しく動作するかを検証します。
* **Feature テスト**: 複数のコンポーネントを組み合わせた機能が正しく動作するかを検証します。

### テストコードの実行

* テストコードは、`php artisan test` コマンドを実行することで実施します。
* テストコードを実行する際に、`- -coverage` をつけることで、カバレッジ率を確認することができます。
    * `php artisan test --coverage`
* `/tests`配下を確認することで、テストコードのファイルを確認することができます。

### テストコードの記述

* テストコードは、`/tests` ディレクトリ以下に作成します。
* Laravel のテスト機能を利用して、テストコードを記述します。
* テストコードの記述方法については、Laravel のドキュメントを参照してください。

## 関連ファイル

* `/tests`: テストコードが配置されているディレクトリ。
    * `/Feature`: Featureテストコードが配置されています。
    * `/Unit`: Unitテストコードが配置されています。
