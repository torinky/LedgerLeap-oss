# 自動リンク機能

## 1. 概要

自動リンク機能は、台帳の本文や台帳定義の説明文に含まれる特定の文字列（例: `SPEC-001`、Redmineチケット番号 `#12345`）を、システム内外の関連情報へ自動的にハイパーリンクに変換する機能です。

これにより、異なる情報間の分断をなくし、システム利用者が情報を参照・追跡する際の利便性を飛躍的に向上させます。

## 2. 機能要件

### 2.1. リンク定義の管理（管理者向け）

管理者は、Filamentで構築された管理画面から、リンク変換のルールを直感的に定義・管理できます。

-   **正規表現によるパターン定義:** リンクに変換したい文字列のパターンを、正規表現を使って柔軟に定義できます。
-   **URLテンプレート:** パターンに一致した文字列の一部（キャプチャグループ）を利用して、動的なリンク先URLを生成できます。（例: パターン `/#(\d+)/` と URLテンプレート `https://redmine.example.com/issues/$1` で、`#12345` を `https://redmine.example.com/issues/12345` に変換）
-   **プレビュー機能:** ルール作成時に、サンプルテキストがどのように変換されるかをリアルタイムで確認でき、正規表現のデバッグを容易にします。
-   **優先順位:** 複数のルールが同じ文字列にマッチする場合、優先度の高いルールが先に適用されます。
-   **適用範囲（スコープ）:** 作成したルールを、特定のフォルダ配下にのみ適用させることができます。これにより、テナントやプロジェクトごとに異なるルールを運用できます。
-   **リンクの種類とアイコン:** リンクの種類（内部リンク、外部リンク、チケットなど）を設定でき、表示時に対応するアイコンが付与されます。これにより、ユーザーはリンクの種類を一目で識別できます。
-   **その他の管理機能:** 定義の有効/無効切り替え、既存定義の複製（クローン）機能、キーワードによる全文検索など、効率的なメンテナンスを支援する機能を提供します。

### 2.2. リンクの自動生成（全ユーザー向け）

台帳レコードの閲覧時や、台帳定義の説明文が表示される際、以下のルールに基づいてテキストが自動的にリンク化されます。

1.  **自動採番カラム（最優先）:** 「自動採番」として設定されたカラムの値は、他のルールに優先して、システム内の台帳を横断検索する特別なリンクに変換されます。
2.  **カスタム定義:** 上記以外の場合、管理者が定義したルールが、優先順位の高いものから順に適用されます。一度リンクに変換された文字列は、後続のルールの対象にはなりません。

### 2.3. テナント横断検索とリダイレクト

自動採番カラムなど、特定のテナントに縛られないIDからリンクが生成された場合、システムは以下のインテリジェントな挙動を示します。

1.  **横断検索:** まず、全テナントを対象に、リンクされたIDを持つ台帳を検索します。
2.  **直接リダイレクト:** 検索結果が1件だけだった場合、ユーザーを直接その台帳の詳細ページへリダイレクトさせます。
3.  **結果リスト表示:** 検索結果が複数件あった場合、または0件だった場合は、検索結果の一覧ページを表示します。

これにより、ユーザーは目的の情報に最短でたどり着くことができます。

### 2.4. 権限管理と監査証跡

-   自動リンク定義の作成・編集・削除は、`manage_auto_links` 権限を持つ管理者のみに許可されます。
-   全ての操作はアクティビティログに記録され、いつ、誰が、どのような変更を行ったかを正確に追跡できます。

## 3. 技術的な実装

-   **リンク変換処理:** `AutoLinkService` が中核を担います。このサービスは、キャッシュ機構と`DOMDocument`を利用して、パフォーマンスとHTML構造の安全性を両立したリンク変換処理を実行します。
-   **管理UI:** `AutoLinkResource` を中心としたFilamentコンポーネント群によって構築されています。
-   **データベース:** `auto_links`（定義本体）と `auto_link_scopes`（適用範囲）の2つのテーブルで構成されます。
