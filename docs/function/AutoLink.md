# 自動リンク機能

## 1. 機能概要

台帳レコードの本文や、台帳定義の説明文に含まれる特定のパターンの文字列（例: Redmineのチケット番号 `#12345`、仕様書番号 `SPEC-001` など）を、システム内外の指定されたURLへのハイパーリンクに自動的に変換する機能です。

この機能により、散在する関連情報へのアクセス性が向上し、ユーザーは情報を探す手間なく、ワンクリックで目的のページに移動できます。

## 2. 管理者向けガイド

自動リンク機能の振る舞いは、管理画面から柔軟に設定できます。

### 2.1. 自動リンク定義の管理

Filamentで構築された管理画面から、リンク変換のルールを定義します。

- **アクセス方法:** [管理画面] -> [自動リンク]
- **必要権限:** `自動リンクを管理できる` (`manage_auto_links`)

### 2.2. 主な設定項目

| 項目 | 説明 | 設定例 |
| :--- | :--- | :--- |
| **テンプレート** | よく使われる設定の雛形を呼び出します。「Redmineチケット」などを選択すると、パターンとURLテンプレートが自動入力されます。 | `Redmineチケット` |
| **ラベル** | この定義を識別するための分かりやすい名前です。管理画面の一覧に表示されます。 | `Redmineチケットへのリンク` |
| **説明** | この定義の目的や補足事項を自由に記述できます。 | `「#」で始まる数字をRedmineのチケットにリンクします。` |
| **パターン** | リンクに変換したい文字列を検出するための**正規表現**です。 | `/#(\d+)/` |
| **URLテンプレート** | 変換先のURL形式です。パターンの正規表現でキャプチャした部分を `$1`, `$2` のように埋め込めます。 | `https://your-redmine/issues/$1` |
| **適用範囲 (フォルダ)** | この定義を特定のフォルダ配下でのみ有効にしたい場合に設定します。設定しない場合は、システム全体で有効になります。 | `開発部/仕様書` |
| **優先順位** | 複数の定義が同じ文字列にマッチした場合の優先度です。**数値が小さいほど優先**されます。 | `10` |
| **有効/無効** | この定義を一時的に無効化できます。 | `有効` |
| **新しいタブで開く** | チェックを入れると、生成されるリンクが新しいブラウザタブで開かれます (`target="_blank"`)。 | `オン` |

### 2.3. プレビュー機能

正規表現やURLテンプレートが意図通りに動作するかを、定義の保存前にリアルタイムで確認できます。

- **テストテキスト:** サンプルの文字列を入力します。
- **変換後プレビュー:** 実際にリンクが生成された結果が表示されます。
- **キャプチャグループ:** 正規表現のどの部分(`$0`, `$1`...)が、どの文字列にマッチしたかをテーブル形式で確認でき、デバッグに役立ちます。

### 2.4. 権限と監査

- **権限:** `manage_auto_links` 権限を持つユーザーのみが、自動リンク定義の作成・更新・削除を行えます。
- **監査証跡:** すべての操作はアクティビティログに記録され、「いつ、誰が、どの定義を、どのように変更したか」を追跡できます。

## 3. 利用者向けガイド

### 3.1. リンクが生成される場所

以下の画面で、管理者が設定したルールに基づいて文字列が自動的にリンクに変換されます。

- 台帳の詳細画面（各カラムの値）
- 台帳の一覧画面（各カラムの値）
- **台帳レコードの複数行入力カラム（Markdown形式）**
- 台帳の新規作成画面（台帳定義の説明文）
- 台帳の詳細画面（台帳定義の説明文）
- 台帳定義のプレビュー画面（台帳定義の説明文）

複数行入力カラムでは、入力されたMarkdownがHTMLに変換された後、そのテキスト部分に対して自動リンクが適用されます。

### 3.2. リンクの優先順位

文字列のリンク化には、以下の優先順位が適用されます。

1.  **（最優先）自動採番カラム:** `auto_number` 型として定義されたカラムの値は、他のどのルールよりも優先され、台帳内検索結果へのリンクになります。
2.  **カスタム定義:** 上記以外の場合、管理者が設定したカスタム定義が適用されます。複数のルールにマッチする場合は、**優先順位**の設定値が最も小さいものが適用されます。
