# 自動リンク機能 マルチテナント対応計画 (方針転換版)

**日付:** 2025年9月15日
**作成者:** Gemini
**ステータス:** 計画中

**関連ドキュメント:**
*   [AutoLink機能概要](/docs/function/AutoLink.md)
*   [AutoLinkService概要](/docs/services/AutoLinkService.md)

## 1. 概要

本計画は、LedgerLeapの自動リンク機能（AutoLink）をマルチテナント環境に対応させるための実装計画である。
当初、閲覧者のコンテキストに応じてリンク先テナントを動的に変更する案を検討したが、要件との不一致が判明した。本計画では方針を転換し、「**自動リンクの定義はシステム全体でグローバルに管理しつつ、リンク先は特定のテナントに固定できる**」機能の実装を目指す。

具体的には、`AutoLink` 定義自体はテナントに依存しないグローバルなリソースとする。しかし、各定義に「リンク先テナント」を任意で紐付けることができるようにし、指定された場合はそのテナントのドメインを含んだURLを生成する。これにより、管理者はシステム共通のルールを作成しつつ、リンク先を柔軟に制御できるようになる。

## 2. 目的

*   `AutoLink` 定義をテナントに依存しないグローバルなリソースとして管理できるようにする。
*   内部リンク（例: `/l/` で始まる台帳リンク）を生成する際に、**リンク先のテナントを明示的に指定**できるようにする。
*   管理画面（Filament）のUIを改善し、テナントを意識した直感的で間違いのない設定を可能にする。

## 3. ペルソナとユーザーシナリオ

### 3.1. ペルソナ

*   **佐藤さん（システム管理者）:**
    *   **背景:** LedgerLeap全体の運用を担当。複数のテナント（企業）がシステムを利用しており、それぞれのテナントに合わせた設定と、システム全体で共通化すべき設定を管理している。
    *   **関心事:** テナントごとに異なる業務ルール（製品コードやプロジェクトIDの体系など）を、自動リンク機能で効率的に吸収したい。設定ミスを防ぎ、メンテナンス性を高く保ちたい。

### 3.2. ユーザーシナリオ

*   **シナリオ1: テナント横断の共通ルール設定**
    *   佐藤さんは、全社共通で使われるRedmineのチケット番号（例: `#12345`）が、どのテナントの台帳に書かれても、常に単一のRedmineサーバーのURLにリンクされてほしいと考えている。

*   **シナリオ2: 特定テナントへのリンク設定**
    *   テナントA（A社）には「A社製品コード（`A-PROD-XXX`）」があり、テナントB（B社）には「B社仕様書ID（`B-SPEC-YYY`）」がある。
    *   佐藤さんは、「`A-PROD-XXX`」という文字列が台帳に現れた場合、**常にテナントAの**特定台帳へのリンクが生成されるように設定したい。このルールはテナントBのユーザーが閲覧している場合でも同様に動作する必要がある。
    *   同様に、「`B-SPEC-YYY`」は**常にテナントBの**台帳へリンクさせたい。

## 4. 機能要件

上記シナリオを実現するため、以下の機能要件を定義する。

1.  **`auto_links` テーブルの修正:**
    *   `tenant_id` カラムを `nullable` で追加する。このカラムは、リンク先のテナントを固定するために使用する。
    *   `tenant_id` が `NULL` の場合は、テナントに依存しないグローバルなリンク（例: 外部サイト）として扱う。

2.  **`AutoLink` モデルの修正:**
    *   `stancl/tenancy` の `BelongsToTenant` トレイトは使用しない（定義はグローバルなため）。
    *   `tenant()` リレーション (`BelongsTo`) を定義し、`auto_links.tenant_id` と `tenants.id` を紐付ける。

3.  **`AutoLinkService` の改修:**
    *   `createCustomLink()` メソッドを修正。`$autoLink->tenant_id` が設定されており、かつ `url_template` が内部リンク形式（例: `/l/$1`）の場合、`tenant_id` で指定されたテナントのドメインをURLに付与するロジックに変更する。

4.  **Filament管理画面 (`AutoLinkResource`) のUI改善:**
    *   **リンク先テナントの選択:**
        *   フォームに「リンク先テナント (`tenant_id`)」を選択するドロップダウンを追加する。
        *   このドロップダウンは、`url_template` が内部リンク形式（例: `/l/` で始まる）の場合にのみ表示されるように動的に制御する。
    *   **フォルダ選択UIの改善:**
        *   適用範囲を設定するフォルダ選択UIで、各フォルダがどのテナントに属するかを識別しやすくする。
        *   ルートフォルダの表示を「**【テナントID】/ フォルダ名**」形式に変更する。
    *   **プレビュー機能の改善:**
        *   プレビュー機能のURL生成ロジックを修正し、「リンク先テナント」で選択されたテナントのドメインを正しく反映するようにする。

## 5. 詳細な実装計画

### ステップ1: データベースとモデルの変更

*   **目的:** `AutoLink` にリンク先テナントを紐付けるための下準備を行う。
*   **タスク:**
    1.  `add_tenant_id_to_auto_links_table` という名前で新しいマイグレーションファイルを作成する。
    2.  `auto_links` テーブルに `tenant_id` カラム（`string`, `nullable`, `index`）を追加し、`tenants` テーブルへの外部キー制約を設定する。
    3.  `AutoLink` モデルの `$fillable` 配列に `tenant_id` を追加する。
    4.  `AutoLink` モデルに `tenant()` リレーション (`belongsTo(Tenant::class)`) を定義する。
*   **検証方法:** マイグレーションが正常に実行され、モデルのリレーションが正しく機能することを確認する。

### ステップ2: リンク生成ロジックの修正

*   **目的:** `AutoLinkService` が、設定された `tenant_id` に基づいて正しいリンクを生成できるようにする。
*   **タスク:**
    1.  `AutoLinkService` の `createCustomLink()` メソッドを修正する。
    2.  `$autoLink->tenant` リレーションが存在する場合、そのテナントのドメインを取得し、URLを生成するロジックを実装する。
*   **検証方法:** ユニットテストを作成し、`tenant_id` がある場合とない場合で、URLが正しく生成されることを確認する。

### ステップ3: 管理画面UIの改修

*   **目的:** Filament管理画面で、直感的かつ間違いなく自動リンクを設定できるようにする。
*   **タスク:**
    1.  **`AutoLinkResource.php` の `form()` メソッドを修正する。**
    2.  `url_template` フィールドの下に、「リンク先テナント (`tenant_id`)」を選択する `Select` フィールドを追加する。
        *   `live()` を使用し、`url_template` が `/l/` で始まらない場合はこのフィールドを非表示にする。
    3.  `folders` を選択する `SelectTree` コンポーネントのクエリを修正し、ルートフォルダにテナントIDがプレフィックスとして表示されるようにする。
    4.  `Preview` セクションの `preview_output` のロジックを修正し、「リンク先テナント」で選択された `tenant_id` に基づいてプレビューURLを生成するように変更する。
*   **検証方法:** 管理画面を操作し、UIが仕様通りに動作することを手動で確認する。

## 6. テスト計画

*   **ユニットテスト:**
    *   `AutoLinkService` の `createCustomLink` メソッドについて、以下のケースをテストする。
        *   `tenant_id` が設定されている場合に、指定されたテナントのドメインを含むURLが生成されること。
        *   `tenant_id` が `NULL` の場合に、テナントのドメインを含まないURLが生成されること。
*   **フィーチャーテスト:**
    *   `AutoLink` の作成・更新時に `tenant_id` が正しく保存されること。
    *   管理画面で、`url_template` の内容に応じて「リンク先テナント」フィールドが表示/非表示されること。
    *   管理画面のフォルダ選択で、テナントIDが表示されること。
    *   管理画面のプレビュー機能が、「リンク先テナント」と連動して正しいURLを生成すること。
