# LedgerLeap プロジェクト ライセンス調査と対応

## 1. 目的

LedgerLeapプロジェクトをMITライセンスで公開するにあたり、利用しているサードパーティライブラリのライセンスを調査し、MITライセンスとの互換性、潜在的なリスク、および必要な対応策を明確にすることを目的としました。

## 2. 調査方法

Laravel Sail環境下で以下のコマンドを実行し、PHPおよびJavaScriptの依存関係のライセンス情報を収集しました。

*   **PHP 依存関係:** `vendor/bin/sail composer licenses`
*   **JavaScript 依存関係:** `npx license-checker --production --json` (事前に `npm install -g license-checker` を実行)

## 3. 調査結果と分析

### 3.1. PHP 依存関係

ほとんどのPHPライブラリは、MIT、BSD-3-Clause、Apache-2.0といった許諾的ライセンスで提供されており、MITライセンスと高い互換性があります。

しかし、以下の2つのライブラリに特記事項がありました。

*   **`ezyang/htmlpurifier`**: `LGPL-2.1-or-later`
    *   **分析**: LGPLはコピーレフトライセンスですが、動的リンク（LaravelプロジェクトでのComposerによる利用がこれに該当）の場合、アプリケーション全体をLGPLにする必要はありません。ただし、ライブラリへの変更を公開する義務や、ユーザーがライブラリを置き換えられるようにする義務など、特定の条件を満たす必要があります。本プロジェクトでは、ライブラリに直接変更を加えていないため、LGPLの条件を満たしていると判断しました。
*   **`nette/schema` および `nette/utils`**: `BSD-3-Clause, GPL-2.0-only, GPL-3.0-only`
    *   **分析**: これらのライブラリは複数ライセンスで提供されており、利用者はBSD-3-Clause（許諾的）またはGPL（コピーレフト）のいずれかを選択できます。GPLを選択した場合、プロジェクト全体がGPLの制約を受ける可能性があります。

### 3.2. JavaScript 依存関係

ほとんどのJavaScriptライブラリは、MITまたはBSD-3-Clauseライセンスで提供されており、MITライセンスと高い互換性があります。

特筆すべきは以下のライブラリです。

*   **`@fortawesome/fontawesome-free`**: `(CC-BY-4.0 AND OFL-1.1 AND MIT)`
    *   **分析**: Font Awesomeは、フォントファイル（OFL-1.1）、CSS/SCSS/LESSファイル（MIT）、アイコンのグラフィック（CC-BY-4.0）でそれぞれ異なるライセンスが適用されています。CC-BY-4.0は「表示（Attribution）」を義務付けるライセンスであり、アイコンを使用していることを明示する必要があります。

## 4. 識別されたリスクと対応策

### 4.1. コピーレフトライセンスの混入リスク

*   **リスク**: `ezyang/htmlpurifier` (LGPL) および `nette/schema`, `nette/utils` (GPLデュアルライセンス) が、MITライセンスでの公開を妨げる可能性がありました。
*   **対応策**:
    *   **`ezyang/htmlpurifier`**: Laravelの通常の依存解決は動的リンクに該当するため、LGPLの条件（変更の非公開、ユーザーによる置き換え可能性）を満たしていると判断し、そのまま利用を継続します。
    *   **`nette/schema`, `nette/utils`**: これらのライブラリは**BSD-3-Clauseライセンスを選択して利用している**ことを明確にすることで、GPLの制約を回避します。

### 4.2. 帰属表示の義務

*   **リスク**: `@fortawesome/fontawesome-free` のCC-BY-4.0ライセンスは、アイコンの利用に対する帰属表示を義務付けています。
*   **対応策**: プロジェクトのどこか（例: フッター、`NOTICE.md` ファイル）に、Font Awesomeを使用している旨と適切な帰属表示を記載します。

## 5. 実施した作業

上記の調査結果と対応策に基づき、以下の作業を実施しました。

1.  **`LICENSE` ファイルの作成**:
    *   プロジェクトのルートディレクトリに `LICENSE` ファイルを作成し、MITライセンスの全文を記載しました。
    *   著作権表示は `Copyright (c) 2025 LedgerLeap Project Contributors` としました。
2.  **`NOTICE.md` ファイルの作成**:
    *   プロジェクトのルートディレクトリに `NOTICE.md` ファイルを作成し、以下の特記事項を記載しました。
        *   `ezyang/htmlpurifier` がLGPL-2.1-or-laterライセンスであり、その利用条件について。
        *   `nette/schema` および `nette/utils` が複数ライセンスであり、BSD-3-Clauseライセンスを選択して利用している旨。
        *   `@fortawesome/fontawesome-free` の帰属表示について。
        *   その他、主要なライブラリのライセンス概要。

## 6. 結論

上記の対応策を実施したことにより、LedgerLeapプロジェクトはMITライセンスで公開することが可能であると判断しました。

## 7. 今後の展望 / 要検討事項

*   **ライセンス情報の継続的な管理**: 新しいライブラリの導入や既存ライブラリのバージョンアップ時には、再度ライセンス互換性を確認するプロセスを確立することが望ましいです。
*   **法務専門家への相談**: 複雑なライセンス問題や判断に迷う場合は、必ずオープンソースライセンスに詳しい法務専門家への相談を推奨します。
