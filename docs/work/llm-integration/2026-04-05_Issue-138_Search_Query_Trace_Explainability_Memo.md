# Issue 138: 検索クエリのトレース可視化と説明可能性

**対象Issue:** `#138`  
**親Issue:** `#136`  
**関連:** `#137`, `#139`

## 0. サマリー

- `#138` は、検索そのものを増やす作業ではなく、**LLM が対話の中でどう検索語を選んだかを説明できるようにする作業**
- 追跡したい最小項目は `original q` / `normalized q` / `selected terms` / `excluded terms`
- `selected terms` には同義語か技術用語かの識別を含める
- 正規化の再実装はしない。半角/全角・空白の揺れは既存前処理に委ねる
- ユーザー向けには短い説明、開発者向けには詳細 trace を分ける
- `#138` の結論は「trace をどう返すかの契約を固定すること」

## 1. 目的

LLM がユーザーとの対話を通じて候補を選んだ場合、後続対話で「どの語で検索したか」を把握できる必要がある。

この memo では、検索クエリの説明可能性を、**ユーザーに返す最小 trace** と **開発者向け詳細** に分けて定義する。

## 2. 既存前提

- 半角/全角、空白などの正規化は既存検索前処理で対応済み
- そのため、`A2-2` では再実装しない
- `SearchContext` / `SynonymService` は既に存在する
- `SearchRequest` 側でも検索前処理がある
- `SearchLedgersTool` は `summary` / `raw` の返却形式を既に持つ

## 3. トレースの最小項目

### 3.1 ユーザー向け最小 trace

| 項目 | 意味 | 例 |
|---|---|---|
| original q | ユーザーが最初に入力した語 | `請求` |
| normalized q | 既存前処理後の語 | `請求` |
| selected terms | 実際に検索へ使った語。`kind` で同義語 / 技術用語を識別する | `請求書 (synonym)`, `インボイス (technical)` |
| excluded terms | 採用しなかった候補。必要なら `kind` と合わせて残す | `売掛 (technical)`, `与信 (technical)` |

### 3.2 開発者向け詳細

- 候補バケットごとの内訳
- 同義語 / 技術用語の kind
- どの候補がどの語から来たか
- どのペルソナ前提で採用されたか
- 必要なら debug log 側で完全版を残す

## 4. 出し分け方針

### 4.1 ユーザー向け

- 短く、後続対話の調整に使える粒度にする
- 「なぜその検索になったか」を1〜2行で説明できるようにする
- 詳細な内部実装名は出さない

### 4.2 開発者向け

- trace の完全版はデバッグ向けに分ける
- LLM に見せる要約と、内部ログは切り離す
- 実装上の都合より、会話の説明しやすさを優先する

## 5. 対話での扱い

1. LLM は会話文脈から候補を選ぶ
2. その結果を `selected terms` として残す
3. 後続対話では、その trace を参照して `q` を調整する
4. 迷う候補は `excluded terms` または保留として残す
5. `selected terms` は `kind` を持ち、同義語か技術用語かを区別できるようにする

## 6. 非ゴール

- 半角/全角揺れの新規実装
- 空白正規化の再実装
- フォルダ名 fragment / 台帳名 fragment
- 添付関連の変更
- 同義語候補の taxonomy 再設計（これは `A2-1`）

## 7. 完了条件

- `original q` / `normalized q` / `selected terms` / `excluded terms` の最小 trace が定義されている
- `selected terms` に同義語 / 技術用語の識別が含まれている
- ユーザー向け要約と開発者向け詳細の二層分離が説明できる
- 後続対話で `q` を調整できる粒度になっている
- 既存の正規化は別レイヤーで対応済みとして扱われている
- `A2-3` に引き継げる粒度になっている

## 8. 次の引き継ぎ

- `#137`: 同義語・技術用語候補の分離と選択ポリシー
- `#139`: 対話文脈からの検索クエリ構成ルールとペルソナ案内

## 9. 結論

`#138` は、検索結果そのものよりも、**LLM がどの語を選んで検索したかを説明できる trace 契約を固定する作業**として完了させる。

この memo をもって `A2-2` の最終アウトプットとする。
