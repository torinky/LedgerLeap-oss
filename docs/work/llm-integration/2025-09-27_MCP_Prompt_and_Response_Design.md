# MCPプロンプトと応答内容の設計案

**日付:** 2025年9月27日

**関連ドキュメント:** [LedgerLeap MCPサーバー実装計画](./2025-09-27_MCP_Server_Implementation_Plan.md)

---

## 1. 概要

本ドキュメントは、`@docs/work/2025-09-27_Docs_Persona_UseCase_Scenario.md` で定義されたペルソナとユースケースに基づき、`gemini` CLIが `@ledgerleap-api` MCPツールを通じてユーザーに応答する際の、具体的な質問応答例と設計方針を定義する。
これは、今後のプロンプトチューニングおよびMCPツール改修のたたき台となることを目的とする。

## 2. MCPの応答内容検討：ペルソナ別のアプローチ

現在のMCPツールは、主にデータベースの情報をJSON形式で返す。これを各ペルソナのニーズに合わせて加工・要約して提示することで、`gemini`の応答価値を最大化する。

### 2.1. 実務担当者 (Operational Staff)

**ニーズ:** 具体的で、タスク指向。シンプルで分かりやすい情報。

#### 質問例1: 「昨日私が作成した日報を見せて。」

*   **Geminiの内部処理:**
    1.  `@ledgerleap-api search_ledgers` を呼び出す。
    2.  パラメータ: `q: "日報"`, `creator_id: <ユーザーID>`, `created_from: <昨日>`, `format: 'summary'`, `include_content: false`
    3.  返されたJSONデータを解析し、一覧形式で提示。

*   **Geminiの応答例（あるべき姿）:**
    > 昨日あなたが作成した日報は1件です。
    >
    > *   **件名:** 2025年9月26日 営業日報
    > *   **ステータス:** 承認待ち (点検完了)
    > *   **更新日時:** 2025-09-26 18:30
    > *   **プレビュー:** 訪問先: A社 / 商談内容: 新製品の紹介を実施...
    >
    > 詳細を確認しますか？

*   **ポイント:**
    *   `format=summary` + `include_content=false` で一覧に最適化
    *   プレビューで概要を把握できる
    *   詳細が必要なら追加リクエストで `include_content=true` を使用
    *   JSONを生で返すのではなく、要点を抜き出してリスト形式で提示

#### 質問例2: 「至急対応が必要な承認待ちのタスクはある？」

*   **Geminiの内部処理:**
    1.  `@ledgerleap-api get_pending_approvals` を呼び出す。
    2.  パラメータ: `format: 'summary'`（担当者IDは認証情報から自動取得）
    3.  返されたデータから期限順にソートし、優先度が高いものを強調。

*   **Geminiの応答例（あるべき姿）:**
    > はい、あなた宛ての承認待ちタスクが3件あります。特に以下の1件は本日が期限です。
    >
    > *   **台帳:** 経費精算申請書
    > *   **申請者:** 営業部 佐藤さん
    > *   **申請日時:** 2025-09-26 15:00
    > *   **件名:** 交通費精算 (9月分)
    > *   **優先度:** 高（期限: 本日）
    >
    > すぐに内容を確認して承認作業を行いますか？

*   **ポイント:**
    *   専用の `get_pending_approvals` ツールを使用（既に実装済み）
    *   「至急」という意図を汲み取り、優先順位をつけて提示
    *   期限・優先度の情報を活用して判断を助ける
    
    #### 質問例3: 「契約書Aの最新版と一つ前の版で、どこが変わったか教えて。」
    
    *   **Geminiの内部処理:**
        1.  `@ledgerleap-api search_ledgers` で "契約書A" を検索（`q: "契約書A"`, `format: 'summary'`, `include_content: true`, `include_attachments: true`）し、対象の台帳IDを特定する。
        2.  `@ledgerleap-api get_workflow_history` を台帳IDで呼び出し、最新と一つ前のバージョンの差分情報を取得。
        3.  各バージョンの内容と添付ファイルを比較し、差分を抽出・要約する。
    
    *   **Geminiの応答例（あるべき姿）:**
        > 契約書Aの最新版と一つ前の版を比較しました。主な変更点は以下の通りです。
        >
        > **ファイル変更:**
        > *   旧: 契約書_v1.pdf (512KB) → 新: 契約書_v2.pdf (530KB)
        >
        > **内容変更:**
        > *   **契約期間フィールド:** 「2025年12月末まで」から「2026年3月末まで」に延長
        > *   **違約金条項:** 新規追加（金額: 契約金額の10%）
        > *   **署名欄:** 「法務部確認印」が追加
        >
        > PDFから抽出されたテキストも確認しますか？
    
    *   **ポイント:**
        *   `search_ledgers` で台帳を特定（`include_content=true` + `include_attachments=true`）
        *   `get_workflow_history` でバージョン履歴取得（既に実装済み）
        *   カスタムフィールドと添付ファイルの変更を統合表示
        *   PDFから抽出されたテキストも活用可能
    
    #### 質問例4: 「請求書の内容を確認したい」
    
    *   **Geminiの内部処理:**
        1.  `@ledgerleap-api search_ledgers` で "請求書" を検索（`q: "請求書"`, `format: 'summary'`, `include_content: true`, `include_attachments: true`）。
        2.  返された添付ファイル情報（PDFから抽出されたテキスト）を解析。
    
    *   **Geminiの応答例（あるべき姿）:**
        > A社からの請求書が3件見つかりました:
        >
        > **1. 2025年9月分請求書** (承認済み)
        > *   **添付ファイル:** 請求書202509.pdf (250KB)
        > *   **金額:** ¥150,000 (税込)
        > *   **内訳:**
        >     - システム利用料: ¥100,000
        >     - オプション機能: ¥50,000
        > *   **支払期限:** 2025年10月末
        >
        > **2. 2025年8月分請求書** (承認済み)
        > ...
        >
        > 詳細を確認したい請求書はありますか？
    
    *   **ポイント:**
        *   PDFから抽出されたテキスト（`content_attached`）を自動解析
        *   金額・内訳・期限などの重要情報を抽出
        *   ファイル名・サイズ・MIMEタイプも表示
        *   追加の詳細確認も可能

### 2.2. 管理者 (Administrator / Manager)

**ニーズ:** 網羅的、統計的、監査可能。サマリーや異常値の可視化。

#### 質問例1: 「先週、システム全体でどれくらいの台帳が作成されたか、種類ごとに集計して。」

*   **Geminiの内部処理:**
    1.  `@ledgerleap-api search_ledgers` を期間指定（`created_from`, `created_to`）と `format='raw'` で呼び出す。
    2.  返された全件のJSONデータから、`ledger_define_id` ごとにカウントを集計する。
    3.  `meta.ledger_defines` から台帳定義名をマッピングする。

*   **Geminiの応答例（あるべき姿）:**
    > 先週 (2025/09/20〜2025/09/26) に作成された台帳は、合計で **58件** です。
    >
    > 内訳は以下の通りです。
    >
    > *   営業日報: 25件
    > *   議事録: 15件
    > *   経費精算申請書: 10件
    > *   その他: 8件
    >
    > 詳細なリストを表示しますか？

*   **ポイント:**
    *   統計情報には `format=raw` で軽量に取得
    *   `meta` 情報から台帳定義名を取得
    *   ユーザーの意図通り、統計情報に加工して提示

#### 質問例2: 「ユーザーAの最近のアクティビティで、何か不審な点はある？」

*   **Geminiの内部処理:**
    1.  `@ledgerleap-api get_activity_log` を `user_id: A` で呼び出す（既に実装済み）。
    2.  返されたログデータを解析し、通常の業務時間外のアクセスや、大量のファイルダウンロード、権限変更などの特異な操作を検出する。

*   **Geminiの応答例（あるべき姿）:**
    > ユーザーAの直近24時間のアクティビティを確認しました。
    >
    > 通常の台帳閲覧・編集操作がほとんどですが、1点確認をお勧めする操作があります。
    >
    > *   **日時:** 2025-09-27 03:15 (深夜)
    > *   **操作:** ファイルの一括ダウンロード (15件)
    > *   **対象フォルダ:** /プロジェクトX/機密資料
    >
    > この操作に心当たりはありますか？ 詳細なログを確認しますか？

*   **ポイント:**
    *   `get_activity_log` ツールを使用（既に実装済み）
    *   「不審な点」という観点で情報をフィルタリング・分析
    *   監査の観点から、具体的な日時や対象を明記
    
    #### 質問例3: 「今月提出されたすべての営業日報のうち、『コンプライアンス』の記載がないものをリストアップして。」
    
    *   **Geminiの内部処理:**
        1.  `@ledgerleap-api search_ledgers` で今月の営業日報総数を取得（`q: "日報"`, `created_from: <今月初日>`, `format: 'raw'`）。
        2.  同じ期間・種類の条件に `q: "コンプライアンス"` を追加して再検索（`format: 'summary'`, `include_content=true`）。
        3.  1の結果と2の結果を比較し、2に含まれていない台帳のリストを作成。
    
    *   **Geminiの応答例（あるべき姿）:**
        > 今月提出された営業日報25件のうち、「コンプライアンス」の記載がなかったものは以下の2件です。
        >
        > *   **提出者:** 営業1課 田中さん (2025-09-15提出)
        > *   **提出者:** 営業2課 鈴木さん (2025-09-22提出)
        >
        > これらの担当者に確認を促しますか？
    
    *   **ポイント:**
        *   全文検索機能を活用し、特定のキーワードが含まれて「いない」文書を特定
        *   `include_content=true` で内容を確認して精度を向上
        *   監査やルール遵守の確認といった管理業務を支援
    
    #### 質問例4: 「特定の契約書がシステム内に存在するか確認」
    
    *   **Geminiの内部処理:**
        1.  `@ledgerleap-api search_ledgers` で添付ファイル名を検索（`q: "業務委託基本契約書"`, `attachment_filename: "業務委託基本契約書"`, `format: 'summary'`, `include_attachments: true`）。
        2.  返された結果から、本文マッチと添付ファイルマッチを区別。
    
    *   **Geminiの応答例（あるべき姿）:**
        > 『業務委託基本契約書』を含む台帳が15件見つかりました:
        >
        > **添付ファイルとして存在 (12件):**
        > *   契約管理/A社関連 - 業務委託基本契約書.pdf (2025-09-15, 512KB)
        > *   契約管理/B社関連 - 業務委託基本契約書_改訂版.pdf (2025-08-10, 530KB)
        > ...
        >
        > **本文に記載のみ (3件):**
        > *   会議議事録 - 「業務委託基本契約書について議論」
        > ...
        >
        > 特定のファイルの内容を確認しますか？
    
    *   **ポイント:**
        *   添付ファイル名での検索に対応
        *   ファイルサイズ・更新日時などのメタデータも表示
        *   本文マッチと添付ファイルマッチの区別
        *   コンプライアンス監査・文書管理を支援
    
    #### 質問例5: 「ストレージ容量の大きいファイルを確認」
    
    *   **Geminiの内部処理:**
        1.  `@ledgerleap-api search_ledgers` でサイズの大きいファイルを検索（`attachment_size_min: 10485760`, `format: 'summary'`, `include_attachments: true`）。
        2.  返された添付ファイル情報を集計。
    
    *   **Geminiの応答例（あるべき姿）:**
        > 10MB以上の添付ファイルを持つ台帳が8件あります:
        >
        > *   **プロジェクトX/設計資料** - 設計図面.dwg (25.3MB)
        > *   **研修資料/2025年度** - 研修動画.mp4 (18.7MB)
        > *   **マニュアル/製品** - 操作マニュアル.pdf (15.2MB)
        > ...
        >
        > **合計容量:** 152.3MB  
        > **ファイルタイプ別:**
        > *   DWG: 45.8MB (3ファイル)
        > *   MP4: 35.2MB (2ファイル)
        > *   PDF: 71.3MB (3ファイル)
        >
        > ストレージ最適化の提案をしますか？
    
    *   **ポイント:**
        *   ファイルサイズでのフィルタ機能
        *   容量の可視化と集計
        *   ストレージ管理・コスト最適化を支援

### 2.3. 開発者 (Developer)

**ニーズ:** 技術的詳細、デバッグ情報、APIの生レスポンス。

#### 質問例1: 「キーワード『テストです』で台帳検索した結果のJSONを整形して見せて。」

*   **Geminiの内部処理:**
    1.  `@ledgerleap-api search_ledgers` を `q: "テストです"`, `format: 'detailed'` で呼び出す。
    2.  返されたJSONをそのまま整形する。

*   **Geminiの応答例（あるべき姿）:**
    > `@ledgerleap-api search_ledgers` を `format=detailed` で実行しました。以下が結果のJSONです。
    >
    > ```json
    > {
    >     "ledgers": [
    >         {
    >             "id": 112,
    >             "tenant_id": "tenantc",
    >             "ledger_define_id": 15,
    >             "content": ["テストです", "..."],
    >             "__display_fields__": { ... },
    >             // ... (完全な情報) ...
    >         }
    >     ],
    >     "total": 1,
    >     "meta": { ... },
    >     "__debug_info__": {
    >         "query_time_ms": 45.2,
    >         "database_queries": 3
    >     }
    > }
    > ```

*   **ポイント:**
    *   開発者向けに `format=detailed` でデバッグ情報も含む完全なデータを提供
    *   どのツールを呼び出したかを明記
    *   生データとデバッグ情報で開発を支援

#### 質問例2: 「台帳定義IDが15のスキーマ定義を教えて。」

*   **Geminiの内部処理:**
    1.  `@ledgerleap-api get-ledger-defines-tool` を呼び出す。
    2.  返されたJSON配列の中から、`id: 15` のオブジェクトを検索し、抽出する。

*   **Geminiの応答例（あるべき姿）:**
    > 台帳定義ID `15` のスキーマ情報は以下の通りです。
    >
    > ```json
    > {
    >     "id": 15,
    >     "tenant_id": "tenantc",
    >     "folder_id": 1,
    >     "title": "sit",
    >     "column_define": [
    >         {
    >             "id": 0,
    >             "name": "ut",
    >             "type": "chk",
    >             // ... (省略) ...
    >         }
    >     ],
    >     // ... (省略) ...
    > }
    > ```

*   **ポイント:**
    *   大量のデータの中から、ユーザーが要求した特定の情報だけを正確に抜き出して提示する。
    *   開発者がデータ構造をすぐに理解できるよう、JSON形式で整形して見せる。
    
    #### 質問例3: 「台帳ID: 123 に添付されている `invoice.pdf` から抽出されたテキストインデックスの内容を見せて。」
    
    *   **Geminiの内部処理:**
        1.  `@ledgerleap-api get-attachment-details-tool` のような新ツールを、台帳ID `123` とファイル名 `invoice.pdf` を指定して呼び出す。（※このツールは新規作成が必要）
        2.  返されたJSONデータから、`content` プロパティ（抽出されたテキストが格納されている）を抜き出して表示する。
    
    *   **Geminiの応答例（あるべき姿）:**
        > 台帳ID `123` の添付ファイル `invoice.pdf` から抽出され、検索インデックスに登録されているテキストは以下の通りです。
        >
        > ```text
        > 株式会社 LedgerLeap
        > ご請求書
        >
        > 品名: システム利用料
        > 金額: ¥100,000-
        > (TikaまたはOCRによって抽出された最終的なテキスト)
        > ```
    
    *   **ポイント:**
        *   ファイルから抽出され、最終的に全文検索の対象となるテキストデータそのものを確認したい、という開発者のデバッグニーズに応える。
        *   複数のエンジン結果を比較するのではなく、DBに保存された「正」のデータを確認するシナリオに修正。
