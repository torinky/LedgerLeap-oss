# Issue #151 更新失敗とリカバリの振り返り

## 📅 日付
2026-04-17

## 🔍 背景
「Documentation-Source Code Synchronization Sprint Plan」の整理中に、既存 issue を更新すべき場面で新規作成へ寄ってしまい、以下のルールを一度外しました。

1. **言語ルール**: 日本語で説明・更新する
2. **更新ルール**: 既存 issue があるなら、まず update で足りるか確認する

## 🧠 学び

### 1. プロセス / アプローチ
- **良かった点:** ユーザーからの即時指摘で、誤った方向に固定される前にリカバリできた
- **悪かった点:** 既存 issue を更新すれば足りるかを十分に確認せず、新規作成側に寄ってしまった
- **改善したい点:** issue 作成の前に、既存 issue の有無と更新可否を先に確認する
- **死んだ道筋:** 既存 issue の修正に、新規 issue を起こすのは分断を増やすだけで非効率だった

### 2. 具体的な技術 / 操作
- **失敗パターン:** 既存 Issue #151 の更新で済む場面で、新規 issue 作成を試みた
- **修正パターン:** `issue_write(method: update, ...)` を優先し、本文は日本語で更新する
- **証跡:** issue 更新のリカバリは成功し、既存 issue の運用で整理できた

## 🛠️ 再発防止
- 新規作成の前に、既存 issue / draft / コメントを確認する
- 既存 issue があるなら、まず update で済むかを判断する
- issue 本文は、セッションの要件言語に合わせて日本語で維持する

## 📝 メモ
- この振り返りは、`github-issue-workflow` の「既存 issue 優先」「日本語維持」のルール強化にもつなげる
