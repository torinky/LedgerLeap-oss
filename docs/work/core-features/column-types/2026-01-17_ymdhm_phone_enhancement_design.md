# 入力タイプ（YMDHM, phone）の改善設計と実装計画

**作成日:** 2026年1月17日  
**対象ステータス:** Issue #45 (継続)

## 1. 背景と目的
`CreateColumn` コンポーネントのリファクタリングに伴い、一部の入力タイプの挙動が現在のユーザーニーズやペルソナのシナリオと乖離していることが判明した。特に「自動的な時刻記録」と「柔軟な電話番号（内線等）の保持」に対する要件を整理し、技術者が迷わず実装・拡張できる設計指針を策定する。

## 2. カラム型別の設計詳細

### 2.1. `YMDHM` (日時型：手動入力およびシステム自動記録)
従来の日時入力機能を維持しつつ、オプション設定によりシステムが自動的にタイムスタンプを記録する「隠し属性」としての挙動を選択可能にする。

#### ユーザーシナリオ
- **手動入力が必要なケース:** 過去の作業記録を遡って入力する場合や、将来の予定日時を指定する場合（従来通り）。
- **自動記録が必要なケース:** 作業完了時に手動で時刻を入力する手間を省き、事実に基づいた時刻（作成時・最終更新時）を正確に残したい場合。

#### 設計方針
- **表示・非表示の切り替え:** `auto_fill_type` オプションが設定されている場合のみ、入力フォーム（作成・編集）で非表示（`isHidden() === true`）とする。未設定の場合は従来通り日時ピッカーを表示する。
- **管理画面でのフィードバック:** 台帳定義（Filament）の設定画面において、`auto_fill_type` を選択した際、「この項目は入力フォームでは非表示になり、システムによって自動入力されます」というヒントを表示する。
- **記録タイミング:** `options` により「新規作成時のみ」または「更新の都度」を切り替え可能にする。
- **データ形式:** Unix タイムスタンプとして保持し、表示時にフォーマットを適用する。

#### `ColumnDefine` 設定項目 (options)
| キー | 型 | 内容 |
| :--- | :--- | :--- |
| `auto_fill_type` | `string` | `none` (手動入力・表示) / `created` (作成時・非表示) / `updated` (更新時・非表示) |
| `format` | `string` | 表示形式（例: `Y-m-d H:i`） |

---

### 2.2. `phone` (柔軟な電話番号型)
従来の「数字とハイフンのみ」から、内線番号や国際番号、注釈を含む柔軟な形式を許容する。

#### ユーザーシナリオ
- **現場リーダー:** 取引先名簿で「03-xxxx-xxxx (内線123)」のように、連絡に必要な情報を欠落させずに保存したい。
- **実務担当者:** スマホからの入力時に `+` や `()` を含めて入力したい。

#### 設計方針
- **バリデーション:** 記号（`+`, `()`, ` ` (スペース)）および「内線」等の文字を一部許容する、より緩やかな正規表現へ見直す。
- **正規化の制御:** オプションにより、保存時に「数字のみにする」か「入力値をそのまま維持する」かを選択可能にする。

#### `ColumnDefine` 設定項目 (options)
| キー | 型 | 内容 |
| :--- | :--- | :--- |
| `normalize` | `boolean` | true の場合、保存時に数字以外を除去する。デフォルト false。 |
| `allow_extension` | `boolean` | true の場合、カッコや「内線」等の特定のキーワードを許容する。 |

---

## 3. 実装計画

### Phase 1: インターフェースとモデルの拡張
1.  **`InputType` インターフェースの拡張**:
    - `isHidden(): bool` メソッドを追加。UIレンダリング時にこのフラグを参照して表示をスキップする。
    - `DateType` は `auto_fill_type` が `created/updated` の場合に `isHidden` で true を返す。
2.  **`DateType` (YMDHM) の修正**:
    - `auto_fill_type` に基づき、現在時刻をセットするロジックを `getDefaultValue()` 等に実装。
    - バリデーションルールを `Y-m-d H:i` に対応。
3.  **`PhoneNumberType` の修正**:
    - `getValidationRules()` の正規表現を緩和。
    - `convertToText()` で `normalize` オプションを参照するように変更。

### Phase 2: コンポーネント、管理画面、サービスの連携
1.  **`CreateColumn` / `ModifyColumn`**:
    - `isHidden()` が true のカラムを `render` メソッドのループから除外。
2.  **`LedgerDefineResource` (Filament)**:
    - `YMDHM` 型のオプション設定フィールドに、非表示化に関する `hint()` または `helperText()` を追加。
    - プレビュー機能がある場合、自動入力設定時は非表示状態を反映する。
3.  **`LedgerService`**:
    - 保存処理の直前で、`YMDHM` 等の「更新時に自動セットが必要なカラム」に対して現在時刻を注入する処理を追加。

### Phase 3: テストによる品質保証
1.  **Unitテスト**: 各 `InputType` クラスでの変換・バリデーションロジックの網羅（内線番号のパース、自動刻みの正確性）。
2.  **Featureテスト**: 隠しカラムがフォームに表示されないこと、保存後に期待したタイムスタンプが入っていることの検証。

## 4. 技術的留意事項
- **`AsColumnArrayJson` 制約**: カラム値のセットは直接配列アクセス（`$ledger->content[$id] = ...`）で行うこと。
- **既存データへの影響**: `YMDHM` の仕様変更により、既存の「手動入力されていた日時データ」が自動更新されてしまわないよう、マイグレーションや変換ロジックで考慮が必要。
