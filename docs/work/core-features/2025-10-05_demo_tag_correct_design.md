### システム設計の確認結果

#### データベーススキーマ

```sql
    id INT PRIMARY KEY,
    ledger_define_id INT,  -- 台帳定義に紐づく
    folder_id INT,
```

#### モデル関係
```php

// Tag::ledgerDefine()
```

### タグの本来の目的（ユーザー指摘）

> **「タグはフォルダーと台帳定義の枠を超えて横串で検索するための機能です」**

#### 意味の解釈

1. **横断検索**
   - 異なるフォルダにある台帳定義
   - 異なる種類の台帳定義（日報、議事録、報告書など）
   - これらを共通のタグで検索可能にする

2. **台帳レコード個別のタグは不要**
   - 台帳レコードごとの分類 → **カラムとして定義**
   - 全文検索で検索可能

```php
        '2025年度営業計画',
```
