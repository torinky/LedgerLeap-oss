## 1. 概要

### 1.1 コンセプト

台帳登録画面で入力内容から事前入力リンクを生成し、クリップボードにコピーすることで効率的な繰り返し登録を実現します。

**特徴:**
- URLをクリップボードにコピー（QRコード化や共有方法はユーザー判断）
- 自動生成値（入力者名、日時、自動採番）は初期値と異なる場合のみURLに含める
- 台帳登録画面だけで完結

### 1.2 基本フロー

```
【URL作成】
台帳登録画面 → 想定値を入力 → 「事前入力リンクを取得」ボタン
→ URLをクリップボードにコピー → 必要に応じてQRコード化やメール送信

【URL使用】
リンクをクリック（またはQRコード読み取り） → ブラウザで台帳作成画面が開く（初期値設定済み）
→ 内容確認・修正 → 登録
```

## 2. 機能要件

### 2.1 URLパラメータ形式

**基本形式:**
```
https://{domain}/{tenant}/ledger/create/{ledgerDefineId}?prefill[{カラムID}]={値}
```

**例:**
```
/ledger/create/5?prefill[1]=EQ-001&prefill[2]=3F空調機A&prefill[5]=正常
```

### 2.2 対応カラムタイプ

| カラムタイプ | 対応 | URL生成時の処理 |
|------------|------|----------------|
| text | ✅ | 入力値をそのまま出力 |
| number | ✅ | 入力値をそのまま出力 |
| YMD / YMDHM | ✅ | **初期値と異なる場合のみ出力** |
| select / chk | ✅ | 選択値をそのまま出力 |
| textarea | ✅ | 入力値をそのまま出力 |
| user_name | ✅ | **初期値と異なる場合のみ出力** |
| auto_number | ❌ | 常に自動生成（URL出力不可） |
| files | ❌ | セキュリティ上非対応 |

### 2.3 自動生成値の扱い

**対象カラム:**
- `user_name`: ログインユーザーから自動生成
- `YMD/YMDHM`: 現在日時を自動設定するカラム
- `auto_number`: 自動採番

**ルール:**
- URL生成時: 初期値と同じ値の場合はURLに含めない
- URL解釈時: パラメータがない場合は自動生成ロジックを実行

**理由:**
- 点検者は毎回変わるため、QRコードに特定の人名を埋め込むと不適切
- 日時も同様に、記録日は実際の作業日であるべき

**例:**
```javascript
// URL生成時の判定
if (column.type === 'user_name' || column.type === 'YMD' || column.type === 'YMDHM') {
    const initialValue = getInitialValue(column);
    if (content[column.id] === initialValue) {
        // 初期値と同じ → URLに含めない
        continue;
    }
}
```

### 2.4 新規カラムタイプ: user_name（入力者名）

**オプション設定:**

```json
{
  "type": "user_name",
  "options": {
    "name_format": "full_name",        // or "family_name_only"
    "org_prefix": "bottom_only",       // or "none", "bottom_3_levels"
    "edit_mode": "overwrite"           // or "append"
  }
}
```

**動作:**
- 新規作成時: ログインユーザー情報から自動生成
- 編集時:
  - `overwrite`: 現在のユーザーで上書き
  - `append`: カンマ区切りで追加（重複チェックあり）

**表示例:**
- フルネーム + 最下層組織: `営業一課 山田太郎`
- 苗字のみ + 組織なし: `山田`
- 複数人（appendモード）: `山田, 佐藤, 鈴木`

**実装:**
- `app/Models/ColumnTypes/UserNameType.php` を新規作成
- 組織階層の取得ロジック
- 編集時の追加/上書きロジック

public function mount(int $ledgerDefineId, array $prefillParams = []): void
{
    $this->prefillParams = $prefillParams;
    // ...
    $this->initColumns();
    $this->applyPrefillParams(); // prefillを適用
    // ...
}

protected function applyPrefillParams(): void
{
    foreach ($this->prefillParams as $columnId => $value) {
        if (array_key_exists($columnId, $this->content)) {
            $this->content[$columnId] = $value;
        }
    }
}

// 事前入力リンク生成メソッド
public function generatePrefillLink(): void
{
    $params = [];
    
    foreach ($this->content as $columnId => $value) {
        if ($this->isEmptyValue($value)) continue;
        
        $column = $this->getColumnById($columnId);
        
        // 自動生成カラムは初期値と異なる場合のみ含める
        if ($this->isAutoGeneratedColumn($column)) {
            $initialValue = $this->getInitialValue($column);
            if ($value === $initialValue) {
                continue; // 初期値と同じなので含めない
            }
        }
        
        // パラメータに追加
        if (is_array($value)) {
            foreach ($value as $item) {
                $params["prefill[{$columnId}][]"] = $item;
            }
        } else {
            $params["prefill[{$columnId}]"] = $value;
        }
    }
    
    // URL生成
    $this->generatedURL = route('ledger.create', [...]) . '?' . http_build_query($params);
    $this->showURLModal = true;
}

protected function isAutoGeneratedColumn($column): bool
{
    return in_array($column->type, ['user_name', 'YMD', 'YMDHM', 'auto_number']);

### 3.3 UserNameType の実装

```php
namespace App\Models\ColumnTypes;

class UserNameType implements InputType
{
    public array $options;
    
    public function __construct(array $options = [])
    {
        $this->options = array_merge([
            'name_format' => 'full_name',
            'org_prefix' => 'none',
            'edit_mode' => 'overwrite',
        ], $options);
    }
    
    public function generateValue(\App\Models\User $user): string
    {
        $name = $this->formatUserName($user);
        $orgPrefix = $this->getOrganizationPrefix($user);
        return $orgPrefix ? "{$orgPrefix} {$name}" : $name;
    }
    
    public function generateEditValue(\App\Models\User $user, ?string $currentValue): string
    {
        $newValue = $this->generateValue($user);
        
        if ($this->options['edit_mode'] === 'append' && !empty($currentValue)) {
            $existingNames = array_map('trim', explode(',', $currentValue));
            if (!in_array($newValue, $existingNames, true)) {
                $existingNames[] = $newValue;
                return implode(', ', $existingNames);
            }
            return $currentValue;
        }
        
        return $newValue;
    }
    
    // formatUserName(), getOrganizationPrefix() など
    // ...
}
```

## 5. QRコード生成ライブラリの選定とトレードオフ

本機能の実装にあたり、候補となったPHP用QRコード生成ライブラリの比較調査結果を以下に示します。

### 5.1 候補ライブラリの比較

| ライブラリ名 | 特徴 | メリット | デメリット |
| :--- | :--- | :--- | :--- |
| **SimpleSoftwareIO/simple-qrcode** | BaconQrCodeのLaravelラッパー (採用) | ・LaravelのFacade/サービスプロバイダが標準提供<br>・SVG/PNG/EPS等主要形式に対応<br>・導入が容易 (composer一本) | ・BaconQrCodeに依存<br>・細かなカスタマイズにはBaconQrCodeの直接操作が必要な場合がある |
| **endroid/qr-code** | モダンで多機能なスタンドアロンライブラリ | ・クリーンなオブジェクト指向設計<br>・ドキュメントが非常に充実<br>・多彩なWriter (SVG, PNG, WebP等) | ・Laravel専用ではないためインテグレーションコードが少々必要 |
| **chillerlan/php-qrcode** | 純粋PHP実装、高速・軽量 | ・外部依存が少なく軽量<br>・非常に多くの出力形式とECCレベルをサポート<br>・カスタマイズ性が極めて高い | ・APIがやや複雑で学習コストが少し高い |

### 5.2 選定理由：SimpleSoftwareIO/simple-qrcode

本プロジェクトでは、以下の理由から **SimpleSoftwareIO/simple-qrcode** を採用しました。

1. **Laravel Integration:** 本プロジェクトの基盤であるLaravelと密結合しており、開発効率が最も高い。
2. **SVG Support:** ベクター形式での出力が容易であり、UI上でのスケーラビリティとダウンロード機能の要件を満たす。
3. **Maturity:** 多くのLaravelプロジェクトでの採用実績があり、信頼性が高い。

### リスク1: 長いURLでQRコードが複雑化

- **対策:** バリデーションで不正値を検出、ログに記録

### リスク3: 自動生成値の判定ミス
- **対策:** 厳密な初期値比較ロジック、十分なテストケース
