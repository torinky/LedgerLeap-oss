# 事前入力リンク用QRコード表示機能 実装計画書

**作成日:** 2026年1月17日  
**ステータス:** 計画中  
**対象コンポーネント:** `Ledger\PrefillLinkModal`

## 1. 目的
台帳登録画面で生成された「事前入力リンク（URLプリフィル）」をQRコードとして表示することで、モバイル端末等からの入力をよりスムーズにし、現場でのペーパーレス運用や効率的なデータ登録を支援する。

## 2. 技術選定
- **ライブラリ:** `simplesoftwareio/simple-qrcode`
  - 理由: Laravelとの親和性が高く、SVG形式での出力が可能なため、Bladeテンプレートへの埋め込みが容易。軽量でメンテナンスも活発。
- **表示形式:** SVG (スケーラブルで解像度に依存しない)

### 2.1 ライブラリ比較とトレードオフ
主要なPHP/Laravel向けQRコード生成ライブラリの比較は以下の通り。

| ライブラリ | 特徴・メリット | デメリット・注意点 | 採用判断 |
| :--- | :--- | :--- | :--- |
| **simplesoftwareio/simple-qrcode** | Laravel向けに開発されたラッパー。Facadeが提供されており、記述が最も簡潔。 | `bacon/bacon-qr-code`に依存。拡張性は内部ライブラリに制限される。 | **本プロジェクトで採用**。Laravel環境での開発効率を最優先。 |
| **endroid/qr-code** | 非常に多機能（ロゴ埋め込み、ラベル追加、詳細な配色設定）。ドキュメントが充実。 | Laravel専用ではないため、Service Provider等の設定が必要（または手動統合）。 | 将来的に高度なデザインカスタマイズが必要になった際の移行候補。 |
| **chillerlan/php-qrcode** | 軽量かつ高速。QRコードの読み取り機能も内蔵。PDFやEPSなど出力形式が豊富。 | Laravel向けの公式な統合パッケージがない。 | 台帳データの読み取り機能（スキャナー）をサーバー側で実装する場合に検討。 |
| **bacon/bacon-qr-code** | PHPにおけるQRコード生成のデファクトスタンダード（多くのライブラリの基盤）。 | 低レイヤーなAPIのため、Laravelで直接扱うにはボイラープレートが増える。 | `simple-qrcode`を通じて間接的に利用。 |

## 3. 実装内容

### 3.1 バックエンド (PHP)
1.  **ライブラリのインストール**
    ```bash
    composer require simplesoftwareio/simple-qrcode
    ```
2.  **QRコード生成ロジックの追加**
    - `App\Livewire\Traits\HandlesPrefillLinks` または専用のサービスを作成し、URLからQRコード（SVG）を生成するメソッドを実装する。
    - 生成されたSVGデータはLivewireコンポーネント経由でフロントエンドに渡す。

### 3.2 フロントエンド (Blade / Alpine.js)
1.  **共通コンポーネントの更新 (`x-ledger.prefill-link-modal`)**
    - モーダル内にQRコード表示エリアを追加。
    - ローディング中はスケルトンまたはスピナーを表示。
    - QRコードのサイズ調整（モバイルで見やすいサイズ、目安 200x200px）。
2.  **ダウンロード機能**
    - QRコードを画像（PNGまたはSVG）としてダウンロードできるボタンを追加。

### 3.3 翻訳キーの追加 (`lang/ja/ledger.php`)
- `prefill.qr_code_title`: "QRコード"
- `prefill.qr_code_description`: "スマホでスキャンして入力画面を開けます"
- `prefill.download_qr`: "QRコードをダウンロード"

## 4. UI/UX デザイン案
- モーダル上部にURLテキストエリア、その直下にQRコードを配置する。
- QRコードは中央寄せにし、周囲に十分な余白を持たせる。
- ダークモード/ライトモードの両方で視認性が高い配色（基本は白背景に黒のセル）を維持する。

## 5. 実施ステップ
1.  **Step 1:** Composerパッケージのインストールと設定。
2.  **Step 2:** QRコード生成ロジックの実装（SVG出力）。
3.  **Step 3:** `prefill-link-modal.blade.php` への組み込み。
4.  **Step 4:** ダウンロード機能の実装（任意）。
5.  **Step 5:** 動作確認（各種モバイル端末でのスキャンテスト）。

## 6. 考慮事項
- **URLの長さ:** 事前入力パラメータが多い場合、URLが非常に長くなり、QRコードの密度が高くなりすぎる可能性がある。
  - 対策: `simple-qrcode` の `size()` や `margin()` を適切に設定する。将来的にURL短縮機能の導入も検討する。
- **セキュリティ:** 生成されたQRコードに機密性の高い初期値が含まれる場合があるため、表示は権限を持つユーザーのみに限定する（既存のリンク生成権限に準ずる）。
