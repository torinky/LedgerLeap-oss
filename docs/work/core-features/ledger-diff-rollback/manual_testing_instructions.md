# 手動テスト手順書: 台帳差分・履歴機能 (Phase 1)

## 目的
Cycle 3 で実装された UX 改善（ポップオーバー、ナッジリンク、タブ同期等）が、各ペルソナの利用シナリオに沿って正しく動作することを確認します。

## テスト環境
- **テナント**: `demo-tenant`
- **ユーザー**: 各ペルソナに応じた権限を持つテストユーザー

---

## 1. 実務担当者シナリオ (Persona: Practitioner)

### シナリオ1：台帳履歴の確認とクイックアクション
1. 台帳詳細画面（[詳細]タブ）を開く。
2. 画面下部の「前バージョンと比較」リンクを確認する。
3. リンクをクリックする。
   - **期待結果**: 「変更箇所を表示」トグルが自動的にONになり、サイドバイサイド形式の比較が表示されること。
   - **理由**: トグルがOFFの状態では、比較対象（targetDiffId）が設定されていても基本情報のみが表示される仕様。
4. 履歴選択が「最新（Ver.X）」と「Ver.X-1」になっていることを確認する。
5. 差分が表示されている際、フッターのボタンが非表示になり、代わりに「すべての履歴・他のバージョンと比較」ボタンが表示されることを確認。

### 1.2 履歴タブへの誘導
1. 「すべての履歴・他のバージョンと比較」ボタンをクリック。
2. 「更新履歴」タブに切り替わり、最新バージョンと直前バージョンのチェックボックスが自動的に選択されていることを確認。
3. 画面下部にその2件の比較結果が表示されていることを確認。

---

## 2. 現場リーダーシナリオ (Persona: Team Leader)

### 2.1 編集者情報の確認 (ポップオーバー確認)
1. 台帳詳細画面のフッター、または履歴一覧の編集者名（Modifier）にマウスをホバーする。
2. ユーザー情報のポップオーバー（名前、メールアドレス、部署、チャットリンク）が表示されることを確認。
3. チャットリンク（Slack 等）をクリックし、リンクが正しいことを確認（設定時のみ）。

### 2.2 バージョン選択の同期
1. 「更新履歴」タブで、最新（Version N）と数回前（Version N-2）の2つのチェックボックスを選択する。
2. 「基本情報」タブに戻る。
3. 「変更箇所のみ表示」が有効になっており、Version N と Version N-2 の比較が表示されていることを確認。
4. フッターの編集者情報が、Version N（左側）と Version N-2（右側）のものが表示されていることを確認。

---

## 3. 管理者シナリオ (Persona: Administrator)

### 3.1 監査・比較の柔軟性
1. 任意の2バージョンを選択し、詳細な差分を確認。
2. 表示レベル（1：必須項目、2：主要項目、3：全項目）を切り替え、表示される項目が正しくフィルタリングされることを確認。
3. 項目数が多い場合に「もっと見る」ボタンや省略インジケータが適切に機能していることを確認。

---

## 4. パフォーマンス確認 (共通)

### 4.1 レスポンス速度
- [ ] ユーザーポップオーバーのホバーから表示までが体感的にスムーズ（目標100ms以内）か。
- [ ] 履歴タブの切り替えや無限スクロールがストレスなく動作するか。

### 4.2 N+1 問題の確認
- [ ] （開発用デバッグバー等が使える場合）1つの差分表示につき大量のクエリが発行されていないか確認。
- [ ] 特にユーザー情報（Modifier）の取得が、ベース版と比較対象版で効率的に行われているか。
