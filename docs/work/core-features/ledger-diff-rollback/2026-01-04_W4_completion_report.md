# W4 実施内容報告：Cycle 1（履歴タブと共通差分表示）

**作成日:** 2026-01-04
**対象:** Ledger Diff Rollback - Cycle 1 (Phase 1)
**ステータス:** 完了

## 1. 目的と概要

本ドキュメントは、台帳詳細画面における「更新履歴タブ」の新設および「共通差分ビューア」のリファクタリング（Cycle 1 / W4）の実施内容を、W3 設計ドキュメントとの整合性と併せて報告するものである。

## 2. 実施内容サマリー

### 2.1 LedgerHistoryManager (W4-1.3, W4-1.4)
- **履歴一覧タイムライン**: 左側に配置。`badge` によるバージョン表示、更新者アバター、更新日時、コメントを含む。
- **無限スクロール**: `x-intersect` (Alpine.js) を活用し、20件単位での非同期ロードを実装。
- **任意比較 UI (トグル選択)**: リスト内の行をクリックすることで、最大2つまで自由に選択可能。選択された2バージョンのうち、新しい方が「基準」、古い方が「比較対象」として自動ソートされます。

### 2.2 LedgerDiffViewer (W4-1.1)
- **共通インターフェース化**: `baseMeta`, `targetMeta` プロパティを導入し、現在値と過去値、あるいは過去値同士の比較が可能な Dumb Component へリファクタリング。
- **基本情報タブへの統合**: 詳細タブでも新しい差分ビューアを使用し、表示ロジックを一本化。

### 2.3 表示状態同期 (W4-1.2)
- **displayLevel 同期**: 親コンポーネントからのイベントを受信し、差分表示レベルを即時更新。
- **開閉状態 (collapsedStates) 管理**: 
  - `Alpine.store('ledgerState')` を導入。
  - 基本情報タブと履歴タブの間で、グループの折りたたみ状態を LocalStorage 経由で同期。

## 4. 既存機能への影響と変更点

リファクタリングにあたり、基本情報（詳細）タブにおいて以下の変更・整理を行った。

- **「差異を表示」トグルの廃止**: 比較機能は履歴タブに集約。
- **3列テーブル化**: スナップショット表示時も「項目名｜過去値｜現在値」のレイアウトに統一（ロジック共有のため）。
- **サーバーサイド状態の排除**: 折りたたみ状態管理を完全にフロントエンド（Alpine）へ移行し、不要な Livewire 通信を削減。

## 6. 指摘事項への対応 (Cycle 1 修正リミテッド)

初回報告後のフィードバックに基づき、以下の修正を実施しました。

- **「差異を表示」トグルの挙動改善**: OFF 時に「以前の内容」列を完全に非表示にし、画面幅を有効活用できるように修正。
- **表示順の入れ替え**: 比較表示の際、左側に「新しい内容」、右側に「以前の内容」が並ぶように順序を入れ替え（Ver.Y <- Ver.X）。
- **比較対象の同期とロジック改善**:
    - [x] 内容に変更がない（ステータスのみ変更など）場合でも、直前の Diff を特定するフォールバックロジックを `LedgerDiffProcessor` に追加。
- **任意の2地点間比較への対応**: `LedgerDiffViewer` および `LedgerContentProcessor` を拡張し、最新版だけでなく過去バージョン同士の差分表示を完全に統合。
- **バージョン選択のトグル化と自動ソート**: 選択順に縛られない直感的な比較インタラクションを実装。
- **表示レベル調整ボタンの追加**: 更新履歴タブのタイムラインヘッダーに配置。
- **堅牢性の向上 (デバッグ部)**: 
    - 実行時の `format() on null` 回避のため、日時取得時にオプショナルチェーンを導入。
    - Livewire の遅延ロード（lazy）時におけるサービスプロパティの初期化エラーと依存関係解決エラーを解消。
    - **コレクションキーイングの修正**: 添付ファイルデータのキーイング（hashedbasename）不整合による表示消失バグを解決。
    - **UI レグレッション修正**: 親要素の `prose-sm` クラスが添付ファイルの `figure` タグに不要なマージンを適用していた問題を、`not-prose` クラスの付与により修正。
    - **表示レベルボタンの安定化**: HTML 属性（name）の衝突回避と `wire:key` の安定化により、選択状態が解除される問題とタブ間での同期不全を解消。
- **細部修正**: バージョン番号表示の更新タイミング不整合、翻訳キー `ledger.form.column_name` の欠落などを修正。

## 8. 残存課題とW2要件とのギャップ (2026-01-05追記)

ユーザーシナリオに基づくW2要件と現在の実装状況（Cycle 1完了時点）を比較した結果、以下の要件が未実装であることが判明しました。これらは本来Cycle 2, 3での実装予定あるいはPhase 1全体での要件ですが、ユーザー体験（特に「実務担当者」および「現場リーダー」）に直結するため、早期の対応が望まれます。

### 8.1 未実装の必須要件 (W2-1.3)

1. **スナップショット表示モード（差分OFF）**
   - **要件:** 実務担当者が「過去の特定時点の内容」をそのまま確認したい（シナリオB）。
   - **現状:** `LedgerHistoryManager` 内で `<livewire:ledger.ledger-diff-viewer :showChanges="true" ... />` とハードコードされており、強制的に差分表示（3カラム）となる。「以前の内容」列を非表示にする手段がない。

2. **編集者詳細情報の表示（所属・連絡先）**
   - **要件:** 現場リーダーが変更の意図を確認するため、編集者に連絡を取りたい（シナリオD）。
   - **現状:** アバターと名前のみの表示。クリック時のポップオーバーや連絡先表示・コピー機能（W4-3.1 task）が実装されていない。

3. **クイックアクションとガイド**
   - **要件:** 実務担当者が「直前比較」等の操作を迷わず行えるようにする（シナリオC対応）。
   - **現状:** ツールチップや「最新と比較」ボタン（W4-3.2 task）が存在しない。

### 8.2 推奨アクション

これらのギャップを埋めるため、Cycle 2/3 のタスクとして以下を優先的に実施することを推奨します。

- **Action 1:** `LedgerHistoryManager` に「差分を表示」トグルを追加し、`showChanges`プロパティを動的に制御可能にする。
- **Action 2:** アバター/名前部分をコンポーネント化または拡張し、クリックで詳細情報を表示するUIを追加する。
- **Action 3:** 履歴リスト上部にクイックアクションボタンを配置する。
