## 1. 目的・背景

W2-1.1、W2-1.2で明らかになった機能要件を踏まえ、ペルソナ別（実務担当者、現場リーダー、管理者）のUX要件を包括的に整理する。既存機能の流用可能性、新規開発機能、必須/任意機能を明確化し、Phase 1の実装範囲を確定する。

### 記載範囲

- ペルソナ別の必須/任意機能の整理
- 既存機能（基本情報タブ、更新履歴画面）の流用可能性
- 新規開発機能の特定と優先度
- ペルソナごとのUI露出制御方針

### 記載しない内容

- 具体的な実装設計（W3-1.1で定義）
- ワークフロー有効時のロールバック仕様（Phase 2以降）

## 2. 前提条件の整理

### 2.1 W2-1.1、W2-1.2で確定した事項

**PM承認済み:**

1. **共通差分コンポーネント**:
   - 承認履歴テーブル形式で表示（バージョン番号、更新日時、更新者、ステータス、コメント）
   - 任意比較UIは全ユーザーに公開（わかりやすいガイド付き）
   - スナップショット表示モード（差分OFF）を提供
   - 検索ハイライト機能（Phase 1ではオプション）

2. **役割分担・文言ポリシー**:
   - 編集者 = `ledgers.modifier`
   - チャットツール連携: リンクで定義できる範囲で実装
   - バージョン番号: `Ver. N` に統一
   - 基本情報タブ: 最新↔直前の即時差分（既存機能維持）
   - 更新履歴タブ: 任意2バージョン間の詳細比較

### 2.2 対象ペルソナ

- **実務担当者**: 日常的な台帳入力・更新を行う
- **現場リーダー**: 部下の更新内容をレビューし、必要に応じてロールバック
- **管理者**: 監査目的で履歴・差分を確認、システム全体の運用管理

## 3. 既存機能の流用可能性分析

### 3.1 基本情報タブの既存機能

| 機能 | 説明 | 流用可能性 | 備考 |
|------|------|----------|------|
| 表示レベル切替 | 非表示/簡略/詳細の切替 | ✅ 流用 | 履歴タブでも同じ設定を引き継ぐ |
| グループ開閉 | カラムグループの折りたたみ | ✅ 流用 | 状態を親コンポーネントと同期 |
| カラムレンダリング | `LedgerContentProcessor` による型別整形 | ✅ 流用 | 差分コンポーネントでも同じロジックを使用 |
| Auto Link | `auto_links` 設定に基づくリンク自動化 | ✅ 流用 | 差分コンポーネントでも適用 |
| タグ表示 | タグチップ表示、空表示時のハンドリング | ✅ 流用 | 差分コンポーネントでも適用 |
| 添付表示 | カラム別リスト、処理ステータス表示 | ✅ 流用 | 過去バージョンの添付も閲覧可能に拡張 |
| 即時差分（最新↔直前） | 前回内容と現在内容の差分表示 | ✅ 維持 | 既存機能として維持、文言のみ調整 |
| 再処理導線 | OCR再処理ボタン等 | ❌ 不要 | 履歴タブでは閲覧のみ |

### 3.2 更新履歴画面の既存機能

| 機能 | 説明 | 流用可能性 | 備考 |
|------|------|----------|------|
| **コア機能** |
| 差分計算 | 2バージョン比較のフィールド差分生成 | ✅ 流用 | `LedgerDiffProcessor` |
| 差分表示 | ハイライト表示（`LedgerDiffViewer`） | ✅ 流用 | 基本情報タブと同じコンポーネント |
| スナップショット表示 | 単一バージョンの内容表示（差分OFF） | ✅ 流用 | 既存実装あり |
| `ledger_diffs` 参照 | スナップショットデータの取得 | ✅ 流用 | 既存のデータアクセスロジック |
| **バージョン選択UI** |
| スライダーUI | スライダーによるバージョン選択 | ✅ 流用 | `ShowDiff` で実装済み |
| バージョン番号表示 | Ver. N 形式で表示 | ✅ 流用 | W2-1.2で統一済み |
| URL直接指定 | `diffId` パラメータで特定バージョン表示 | ✅ 流用 | 既存実装あり |
| **ワークフロー情報表示** |
| ステータス表示 | `WorkflowStatus` のラベル・カラー | ✅ 流用 | カード形式で実装済み |
| 関連ユーザー表示 | 更新者、検査者、承認者 | ✅ 流用 | カード形式で実装済み |
| 日時表示 | 作成日時、更新日時 | ✅ 流用 | カード形式で実装済み |
| コメント表示 | ワークフロー関連コメント | ✅ 流用 | カード形式で実装済み |
| **その他** |
| 編集者・更新日時表示 | フッターに表示 | ✅ 流用 | 既存実装あり |
| カラムレンダリング | `LedgerContentProcessor` | ✅ 流用 | 基本情報タブと同じロジック |
| 添付ファイル表示 | 過去バージョンの添付も表示 | ✅ 流用 | `setAttachedFilesFromContent` |

## 4. 新規開発機能の特定

### 4.1 Phase 1で新規開発が必要な機能

| 機能 | 説明 | 優先度 | 対象ペルソナ | 備考 |
|------|------|--------|------------|------|
| 承認履歴テーブル | バージョン一覧を表形式で表示、比較対象をハイライト | High | 全員 | 現状はカード形式 |
| 任意2バージョン比較UI | 履歴一覧から任意の2バージョンを選択して比較 | High | 全員（ガイド付き） | スライダーは単一バージョンのみ |
| 更新履歴タブへの統合 | 詳細画面の更新履歴タブに差分コンポーネントを埋め込み | High | 全員 | 遅延ロード対応 |
| 検索ハイライト | 検索キーワードのヒット箇所を強調 | High | 全員 | 基本情報タブに実装あり、流用 |
| 編集者情報表示・コピー | 編集者の所属組織、連絡先を表示・コピー | High | 現場リーダー、管理者 | チャットツール不使用者向け |
| チャットツールリンク | 編集者名からチャットツールへのディープリンク | Medium | 現場リーダー、管理者 | ユーザー設定で定義、オプション |
| クイックアクションボタン | 「最新と比較」「直前と比較」ボタン | Medium | 実務担当者 | 任意比較UIの補助 |
| 初回利用時ガイド | ツールチップで使い方を説明 | Medium | 実務担当者 | ツールチップで対応 |

### 4.2 Phase 2以降で検討する機能

- ロールバック機能（ワークフロー無効台帳のみ）
- ワークフロー有効時の承認者情報表示
- 高度なフィルタ/ソート機能（監査向け）
- 差分エクスポート機能

## 5. ペルソナ別UX要件

### 5.1 実務担当者

#### 主要ユースケース

1. **直近修正内容の確認**（シナリオA）
   - 導線: 基本情報タブの即時差分
   - 必須機能: 既存の即時差分機能（維持）
   - 任意機能: なし

2. **過去の特定更新の内容確認**（シナリオB）
   - 導線: 更新履歴タブ → スナップショット表示
   - 必須機能: 承認履歴テーブル、スナップショット表示モード
   - 任意機能: 検索ハイライト

3. **誤更新時の確認**（シナリオC変形）
   - 導線: 更新履歴タブ → 任意比較（ガイド付き）
   - 必須機能: 任意比較UI、クイックアクションボタン、初回利用時ガイド
   - 任意機能: なし

#### UI露出制御

- **デフォルト表示**: 承認履歴テーブル、スナップショット表示、クイックアクションボタン
- **ガイド付き表示**: 任意比較UI（初回利用時にツールチップ表示）
- **非表示**: 高度なフィルタ/ソート（Phase 2以降）

#### 懸念と対応

- **懸念**: 任意比較UIが複雑で使いこなせない
  - **対応**: クイックアクションボタン（「最新と比較」「直前と比較」）を提供し、よくある操作を簡単に実行できるようにする
  - **対応**: 初回利用時にツールチップで使い方を説明

---

### 5.2 現場リーダー

#### 主要ユースケース

1. **部下の更新内容レビュー**（シナリオD）
   - 導線: 更新履歴タブ → 承認履歴テーブル → 差分表示
   - 必須機能: 承認履歴テーブル（編集者・編集日時表示）、差分表示
   - 任意機能: チャットツールリンク

2. **誤更新のロールバック**（シナリオE - Phase 2）
   - 導線: 更新履歴タブ → ロールバック確認 → 実行
   - 必須機能: ロールバック機能（Phase 2）
   - 任意機能: なし

3. **変更意図の確認**
   - 導線: 承認履歴テーブル → 編集者名クリック → チャットツールリンク
   - 必須機能: 編集者情報表示（`ledgers.modifier`）
   - 任意機能: チャットツールリンク

#### UI露出制御

- **デフォルト表示**: 承認履歴テーブル、任意比較UI、チャットツールリンク
- **権限制御**: ロールバック機能（Phase 2、フォルダ権限・ロールで制御）
- **非表示**: なし

#### 懸念と対応

- **懸念**: 変更意図の確認に時間がかかる
  - **対応**: チャットツールディープリンクで即座に連絡できるようにする
  - **対応**: 承認履歴テーブルにコメント欄を表示し、更新理由を確認できるようにする

---

### 5.3 管理者

#### 主要ユースケース

1. **監査目的での履歴・差分確認**（シナリオF）
   - 導線: 専用履歴画面 → フィルタ/ソート → 任意比較
   - 必須機能: 承認履歴テーブル、任意比較UI、検索ハイライト
   - 任意機能: 高度なフィルタ/ソート（Phase 2）

2. **ロールバック履歴の監査**（シナリオG - Phase 2）
   - 導線: 専用履歴画面 → ロールバックイベント絞り込み
   - 必須機能: ロールバックイベントの表示（Phase 2）
   - 任意機能: 操作ログとの突合機能

#### UI露出制御

- **デフォルト表示**: すべての機能を公開
- **権限制御**: なし（管理者はすべての機能にアクセス可能）
- **非表示**: なし

#### 懸念と対応

- **懸念**: 大量履歴の確認に時間がかかる
  - **対応**: ページング/無限スクロールを実装
  - **対応**: 検索ハイライトで重要な変更箇所を素早く特定

## 7. Phase 1実装範囲の確定

- **現場リーダー**: 編集者情報表示・コピー
