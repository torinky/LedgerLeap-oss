**焦点:** キー・バリューペア検出、表構造認識

## 調査結果サマリ

### 結論

**`use_doc_unwarping`の有無による出力品質の差異:**
- **テキストブロック数:** 同一（69個）
- **Key-Valueペア数:** 同一（5個）
- **信頼度スコア:** 同一

- **唯一の差異:** 処理時間のみ（±1.5秒程度）

**結論:** このテスト画像（invoice_simple.pdf）では、`use_doc_unwarping`の有無による出力品質の違いは**観測されませんでした**。

## 表構造認識テスト

### テストファイル複数での検証

| ファイル | 説明 | Text Blocks | Key-Value | Tables | 備考 |
|---------|------|-------------|-----------|--------|------|

| receipt_01.jpg | 領収書 | 30 | 1 | **0** | - |
| meeting_notes.pdf | 議事録（表あり） | 143 | 0 | **0** | 表の構造は検出されず |

### 表構造認識の制約

**PaddleOCR 2.8 (stable)の制約:**
1. ❌ **表の構造認識は未対応**
   - `tables`フィールドは常に空配列
   - テキストは行単位で抽出されるが、セルの関係性は保持されない
   
2. ✅ **表内のテキスト抽出は可能**
   - 各セルのテキストは`text_blocks`として抽出される
   - ただし、どのテキストがどのセルに属するかの情報はない

3. ⚠️ **表構造が必要な場合の代替案**
   - **PaddleOCR-VL**: PP-StructureV3による表構造解析（GPU推奨）
   - **Marker/MinerU**: Markdown形式で表構造を保持

## Key-Valueペア検出の仕組み

### 検出ロジック

現在の実装では、以下の条件でKey-Valueペアを自動検出:

```python
# 実装コード（unified_api.py）
if ':' in text or '：' in text:
    parts = text.replace('：', ':').split(':', 1)
    if len(parts) == 2:
        key = parts[0].strip()
        value = parts[1].strip()
        if key and value:
            key_value_pairs.append({
                "key": key,
                "value": value,
                "confidence": float(confidence),
                "bbox": [[float(p[0]), float(p[1])] for p in bbox]
            })
```

### 検出条件
1. テキストに`:` または `：` が含まれる
2. コロンで分割して2つの部分に分かれる
3. キーと値がどちらも空でない

### 検出例

**検出される:**
- `請求書番号:00000000` → キー: "請求書番号", 値: "00000000"
- `発行日：0000年00月00日` → キー: "発行日", 値: "0000年00月00日"
- `TEL: 0353216212` → キー: "TEL", 値: "0353216212"

**検出されない:**
- `株式会社` （コロンなし）
- `10時～12時` （コロンの代わりに波ダッシュ）
- `:` （キーまたは値が空）

### 改善の可能性

より高度なKey-Value検出を実現するには:
1. **PaddleOCR-VL**: Key Information Extraction (KIE) 機能
2. **カスタムロジック**: 位置関係やレイアウトを考慮した検出
3. **LLM連携**: 抽出テキストからLLMで構造化データを生成

## use_doc_unwarpingの効果について

### 今回のテスト結果

**invoice_simple.pdf（高品質PDF）での結果:**
- 出力品質: **差異なし**
- 処理時間: 約1.5秒遅くなる

### 効果が期待できるケース

`use_doc_unwarping`が有効に働くのは:

1. ✅ **歪みのある画像**
   - スマホで斜めから撮影した文書
   - スキャナーの設置ミスによる歪み
   - 本や雑誌などの湾曲した面

2. ✅ **パースペクティブ歪み**
   - 角度のついた撮影
   - 遠近感のある画像

3. ❌ **高品質PDF** ← 今回のテスト
   - 既に正確に整形された文書
   - 歪みがない場合は効果なし

### 推奨設定

**結論:**
- 歪みのある文書を扱う可能性がある場合: `use_doc_unwarping=True` を推奨
- 処理時間への影響は限定的（1-2秒程度）
- 高品質PDFのみの場合: 無効化してもよい

**LedgerLeapの推奨:** 
- ユーザーが様々な品質の画像をアップロードする可能性があるため、`use_doc_unwarping=True` を維持することを推奨

## まとめ

### Key-Valueペア検出について

**現状:**
- ✅ コロン区切りのデータは自動検出可能
- ✅ 信頼度スコア付きで出力
- ✅ 座標情報（bbox）も取得可能
- ⚠️ 複雑なフォームには対応困難

**改善策:**
- PaddleOCR-VLのKIE機能を使用
- LLM連携による高度な構造化

### 表構造認識について

**現状:**
- ❌ PaddleOCR 2.8では表構造認識は**未対応**
- ✅ 表内のテキストは抽出可能（構造情報なし）

**代替案:**
- PaddleOCR-VL（PP-StructureV3）
- Marker/MinerU（Markdown形式）

### use_doc_unwarpingについて

**効果:**
- 歪みのある画像での精度向上
- 高品質PDFでは差異なし
- 処理時間は1-2秒増加

**推奨設定:** `True`（様々な品質の入力に対応するため）
