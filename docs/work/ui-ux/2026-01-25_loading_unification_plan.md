# ローディング表現の全域統一化計画 (Issue #53)

## 1. 目的
LedgerLeap の Livewire ビュー全域において、バラバラになっているローディング表現（スピナー、スケルトン、ドット、透過表示）を標準化し、最高品質のデザイン一貫性と操作フィードバックを実現する。

## 2. デザイン標準（3ティア方式）

### Tier 1: フルコンテント・ロード (Main Content)
- **用途**: 画面初期ロード、メインエリアの完全なリフレッシュ。
- **デザイン**: 
  - 構造的スケルトン（`animate-pulse`）
  - `bg-base-300/40` + `backdrop-blur-sm` によるオーバーレイ
  - 中央に大型スピナー (`loading-lg text-primary`) + 「読み込み中...」ラベル

### Tier 2: セクション・ロード (Component Level)
- **用途**: テーブルの検索/フィルタリング、サイドパネルの追加読込。
- **デザイン**: 
  - 該当するカードやパネル内のみを透過オーバーレイ
  - 中央または下部に中型スピナー (`loading-md`)
  - ナビゲーションや他コンポーネントの操作を阻害しない (`wire:target` の活用)

### Tier 3: マイクロ・ロード (Interaction Level)
- **用途**: ボタン操作、インライン検索、トグル切り替え。
- **デザイン**: 
  - ボタン内小型スピナー (`loading-xs`)
  - 入力欄右端のインライン型スピナー

## 4. 技術仕様
- **UI Framework**: MaryUI (DaisyUI) + Tailwind CSS
- **Livewire**: `wire:loading`, `wire:target` を全箇所で明示。
- **UX**: 短時間通信での「点滅」を回避するため、CSS トランジションまたは Livewire の `.delay` 指定を活用。
