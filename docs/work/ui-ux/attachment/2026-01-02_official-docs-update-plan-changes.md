## 3. 新メトリクス追加方法の詳細化

### Before（変更前）
- 簡単なサンプルコードのみ
- 3ステップの簡易ガイド

### After（変更後）
**7ステップの詳細ガイド:**

1. **config/ledgerleap.phpに新メトリクスを追加**
   - 環境変数でON/OFF可能な設計
   - デフォルトfalseで追加（段階的導入）

2. **.envに環境変数を追加**
   - 開発/ステージング/本番の推奨設定を明記

3. **フロントエンドに測定コードを追加**
   - 条件分岐付きのBladeテンプレート実装
   - Alpine.jsでのイベントベース測定
   - `@if($performanceEnabled && $metricEnabled)` パターン

4. **Livewireメソッドで受信**
   - 既存のlogPerformance()が自動対応
   - 追加実装不要

5. **閾値アラートの追加（オプション）**
   - checkThreshold()メソッドの実装例
   - Slack通知等への拡張方法

6. **メトリクスをドキュメント化**
   - operations/fileinspector-performance-monitoring.md更新
   - 測定結果記録テンプレート追加

7. **テストとバリデーション**
   - 設定確認、実測定、ログ確認の3ステップ
   - 実装チェックリスト（7項目）

## 5. 重要な知見の記録

### npm run buildの重要性

**WBS 5.2で判明した問題:**
- npm run dev（開発モード）で測定すると誤った結果が出る
- Viteのオーバーヘッドが問題を隠蔽・誇張

**解決:**
- **必ず本番モード（npm run build）で測定**
- この重要性を両ドキュメントで強調

### wire:ignoreの落とし穴

**実証された問題:**
- MaryUIコンポーネントと相性が悪い
- clearable、money、エラー表示等が破壊される

**対策:**
- 慎重な使用を推奨
- 失敗事例をドキュメント化
