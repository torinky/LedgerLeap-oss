**影響:**

**解決策:** wire:ignoreでLivewireのレンダリングをスキップ

## 📋 WBS 5.2.2 実装内容（次のタスク）

### 目的

キーワード検索を1500ms → <50msに改善

### 実装方針

**wire:ignoreでLivewireのレンダリングをスキップ**

#### Before（現在）

```blade
<!-- content.blade.php -->
<div>
    <input wire:model.live.debounce.1000ms="searchKeyword" />
    <div>{{ $previewText }}</div>
</div>
```

**問題:**
- 検索のたびにLivewireがサーバーリクエスト
- Blade全体を再レンダリング（1500ms）
- UIがブロックされる

#### After（実装予定）

```blade
<!-- content.blade.php -->
<div wire:ignore x-data="{
    previewText: @js($this->getPreviewText()),
    keyword: @entangle('searchKeyword'),
    keywords: [],
    
    init() {
        this.$watch('keyword', (value) => {
            this.keywords = value.trim().split(/\s+/).filter(k => k);
        });
    },
    
    get highlightedText() {
        if (!this.keywords.length) {
            return this.previewText;
        }
        
        let text = this.previewText;
        this.keywords.forEach(keyword => {
            const regex = new RegExp(`(${keyword})`, 'gi');
            text = text.replace(regex, '<mark class=\"bg-warning/40\">$1</mark>');
        });
        return text;
    }
}">
    <input x-model="keyword" type="text" class="input input-sm" />
    <div x-html="highlightedText" class="prose"></div>
</div>
```

**効果:**
- Livewireリクエストなし
- Alpine.jsのみで検索処理
- 1500ms → <50ms ✅
- UIがブロックされない ✅

### 実装手順

1. **content.blade.phpの検索部分を修正**
   - wire:ignoreで囲む
   - Alpine.jsでハイライト処理を実装
   - searchKeywordはentangleで同期

2. **テスト**
   - ブラウザコンソールでパフォーマンス確認
   - ハイライトが正しく動作するか確認

3. **測定**
   - performance.now()で実測
   - Laravelログに記録
