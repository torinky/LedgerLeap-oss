# Issue Draft: タブ再表示時の再読込抑止とファイルインスペクターのタブ別ローディング改善

**作成日**: 2026-03-21  
**対象**: 台帳詳細画面 / ファイルインスペクター  
**関連候補**: `resources/views/livewire/ledger/show.blade.php` / `app/Livewire/Ledger/Show.php` / `resources/views/livewire/attached-file/file-inspector.blade.php` / `app/Livewire/AttachedFile/FileInspector.php`

## 概要

台帳詳細画面とファイルインスペクターで遅延ローディングを導入したが、**一度開いたタブへ戻るたびに再度ローディングが発生する**。  
また、**ファイルインスペクターの各タブでローディング中であることが分かりづらい**ため、ユーザーが「まだ読み込み中なのか」「操作が無反応なのか」を判別しにくい。

この Issue では、以下を同時に満たす UI/状態管理の見直しを行う。

1. 一度読んだタブは再訪時に再初期化しない
2. ファイルインスペクターの各タブに、意図が伝わるローディング表示を追加する
3. 初回表示と再訪表示の役割を分け、視覚的なノイズを減らす

## 現状の問題

### 1. 台帳詳細画面
- タブを切り替えるたびに、既に見たタブでも再度ローディングが出る。
- `selectedTab` を起点に再描画されるため、ユーザーには「毎回読み直している」ように見える。

### 2. ファイルインスペクター
- 初回のドロワー表示には全画面スケルトンがある。
- しかし、`content / details / history / permissions` の各タブには、初回表示後の再訪時に状態を保持する仕組みが弱い。
- タブごとのローディング表示も十分ではなく、何を待っているのかが分かりづらい。

## 期待する状態

- 一度表示したタブは、別タブへ移動して戻っても、不要な再ロードや再初期化をしない。
- ファイルインスペクターでは、タブごとの初回ロード中に適切なスケルトンまたはローディングを表示する。
- 再訪時は、前回の状態を保ったまま即座に表示される。
- 初回ロード、タブ切替、内部の再取得処理がそれぞれ見分けられる。

## 変更方針の候補

### 案 A: 読み込み済みタブを Livewire 側で保持する
- `loadedTabs` のような配列で、開いたタブを記録する。
- 既読タブは再描画時に内容を保持する。
- UI 側では、未読タブにのみローディングを出す。

**利点**
- 実装がシンプル
- 再訪時の UX が安定する

**注意点**
- Livewire 再描画時に、ローディング対象と内容の切り替えを丁寧に設計する必要がある

### 案 B: タブ内容を DOM で保持し、表示/非表示だけ切り替える
- 初回に描画したタブを破棄せず、以後は保持する。
- 再訪時は Livewire の再取得ではなく、既存 DOM を活かす。

**利点**
- 最も「戻ったときに速い」体験になりやすい

**注意点**
- DOM が増える
- タブ状態の同期と hidden 表示の整合が必要

### 案 C: 初回ロードと再訪ロードで表示を分ける
- 未読タブはスケルトン、既読タブは即時表示。
- ローディング表示はタブごとに局所化する。

**利点**
- ユーザーに意図が伝わりやすい

**注意点**
- キャッシュの保持と組み合わせる前提が必要

## 推奨スプリント

### Sprint 1: タブキャッシュの基盤整備
- `Show` / `FileInspector` に、読み込み済みタブを追跡する状態を追加
- 再訪時に既読タブを再初期化しないようにする
- タブ切替イベントと状態管理を整理する

### Sprint 2: ローディング表示の整理
- ファイルインスペクターの各タブに、初回ロード用のスケルトンまたはローディングを追加
- 台帳詳細画面のタブ切替ローディングを、既読タブと未読タブで出し分ける
- 既存の初回ドロワー表示と干渉しないように調整する

## 受け入れ条件

- [ ] 台帳詳細画面で、一度開いたタブへ戻っても不要な再ローディングが起きない
- [ ] ファイルインスペクターの各タブで、初回ロード時にローディング意図が分かる
- [ ] 既読タブは再訪時に表示状態を保持する
- [ ] 初回ドロワー表示とタブ切替ローディングが競合しない
- [ ] 既存の権限表示や履歴表示が崩れない

## 確認メモ

- `selectedTab` の更新だけでなく、「そのタブが既に読まれているか」を別に持つ必要がある可能性が高い。
- ファイルインスペクターは、初回ドロワー用スケルトンとタブ内ローディングの役割分担を明確にする。
- 実装前に、`x-mary-tabs` が DOM を保持する前提かどうかを確認しておくと安全。
