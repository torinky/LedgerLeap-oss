# Issue #53: 台帳リスト画面の UI 調整とローディング改善

## 概要
台帳リスト画面において、z-index の競合解消、ローディング表示範囲の最適化、およびサイドドロワーとの連携時の視覚的フィードバック向上を行う。

## 実施済みの変更

### 1. UI 階層 (z-index) の適正化
- `docs/development/ui-z-index-hierarchy.md` に基づき、要素の重なり順を調整。
- 固定ヘッダー（ナビゲーション）を `z-30` 以上とし、スティッキーな検索パネルなどを `z-10` に設定。
- `loading-overlay` の Tier 1（全画面）を `z-[110]`、Tier 2（セクション）を `z-30` に更新。

### 2. ローディングオーバーレイの改善 (`loading-overlay.blade.php`)
- スピナーが常に画面中央に配置されるよう、CSS (Flexbox, min-h-screen, m-auto) を強化。
- バックドロップの不透明度とブラーを Tier 別に最適化 (Tier 1: `backdrop-blur-md`, Tier 2: `backdrop-blur-[2px]`)。
- **デザインの簡素化**: スピナー背後の Glow 効果（発光）を削除し、メッセージ表示用のピル型ボックスも廃止。テキストのみのクリーンな表示に統一。

### 3. レコード表示・詳細画面の表現統一
- **詳細画面 (`show.blade.php`) の非ブロッキング化**: タブ切り替え時の画面全体を覆う Tier 1 ローディングを廃止。代わりに各タブのコンテンツ内のみに Tier 2 スピナーを表示することで、ヘッダーのインタラクティブ性を維持。
- **表示レベル切り替えの局所化**: 表示レベルの変更時、フィルターやステータス表示を隠さず、実際のレコード表示部分 (`ledger-diff-viewer`) の周りのみがローディング状態になるよう調整。
- **子コンポーネント内へのローディング導入**: `ledger-diff-viewer` (子) が親からのイベントを受けて再描画される際、子コンポーネント自体がローディングを表示するようにし、視覚的な空白時間を解消。`:delay="false"` により即座にスピナーを表示。
- **不要なボックスの除去**: 台帳差分ビューアのプレースホルダーやファイルインスペクターのスケルトン等から、余計な白いボックス（カード枠）を削除し、コンテンツに透過して重なるスピナーのみのデザインに統一。
- **表示レベル切り替えの改善**: 表示レベル選択ボタンをローディング範囲から除外し、常に操作可能な状態を維持（リスト画面・詳細画面共通）。
- **パンくずリストの維持**: 台帳選択・解除時のローディング中にパンくずリストが消えないよう、コンテナ構造を分離。
- **バッジ表示の復元**: `develop` ブランチのデザインに基づき、検索結果件数（フォルダ、台帳、レコード）をバッジ形式 (`badge-info`) に戻し、視認性を向上。

### 4. 新規登録・編集画面のアコーディオン最適化
- **Alpine.js + @entangle への移行**: `ModifyColumn`, `CreateColumn`, および `LedgerDefine/Edit` の各アコーディオンを Alpine.js 制御に刷新。
- **グローバルローディングの抑制**: サーバー側のメソッド (`toggleGroup`, `toggleDescriptionGroup`) に `#[Renderless]` 属性を付与し、かつ `@entangle` を用いることで、単なる開閉操作での Tier 0 プログレスバー発生を完全に排除。
- **スムーズなアニメーション**: DaisyUI の `collapse-open` / `collapse-close` クラスを利用し、CSS ベースの滑らかな展開・折りたたみアニメーションを実現。
- **(修正済み) JavaScript エラーと論理の不整合の解消**: Alpine 移行時に発生した `groupErrorBadge` 未定義エラーと、`isCollapsed` 真偽値の不整合（逆転）を修正。`ledgerEdit.js` にて関数を共通化し、堅牢性を向上。

## 未解決の課題・継続案件

### 1. ツリードロワー操作時のグローバルローディング不具合
- サイドドロワー（Tree コンポーネント）でフォルダを切り替えた際、メインコンポーネント（RecordsTable）側の Tier 1（全画面スピナー）が期待通りに発火しない。
- 現状ではドロワー内のみがローディング状態になり、メイン側はイベント受け取り後のスケルトン表示まで「空白の時間」が発生している。
- Livewire 3 におけるコンポーネントを跨いだローディング状態の伝播について、更なる調査または Alpine.js / ブラウザイベント等を用いた別のアプローチが必要。

### 2. スピナーの挙動
- 特定の操作条件下で、スピナーが垂直中央からずれて上方に張り付くケースが依然として報告されている（CSSのさらなる堅牢化が必要）。
