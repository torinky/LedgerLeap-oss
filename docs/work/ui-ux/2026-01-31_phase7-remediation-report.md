# Phase 7: リアクティブ統合とドロワーUIの是正レポート

**作成日**: 2026年1月31日  
**関連Issue**: #53, #102（IndexManagerのリアクティブ同期）  
**フェーズ**: Phase 7（リアクティブ統合・ドロワー連携）

## 概要
台帳リスト画面 (`IndexManager`) を中心とした、親子コンポーネント間のリアクティブな状態制御の導入と、それに伴うローディング不具合（スケルトンの不一致、スピナーの位置ずれ）の解消を目的とした是正作業。

## 課題と解決策

### 1. [継続中] ドロワーからのフォルダ遷移不具合
**現象**:
- ツリードロワー (`Folder/Tree`) のフォルダをクリックしても、メインエリア (`RecordsTable`) の内容が更新されない、または更新されてもスケルトンが表示されない。

**解決の変遷**:
1. **イベント発火方式**: `currentFolderChangeRequested` イベントを dispatch。通信は確立するが、メインエリアの `wire:target` がヒットせずスケルトンが出ない。
2. **$parent 方式**: `wire:click.prevent="$parent.changeCurrentFolder"` で親のメソッドを直接叩く試行。再帰構造により `$parent` の参照が不安定になり、クリック自体が不発になるケースが発生。
3. **Alpine.js dispatch**: `$dispatch` を使い DOM イベントとして上位へ通知。最も安定して通信は行えるが、やはり `IndexManager` 側のスケルトン連動が弱い。
4. **[採用] Livewire.find().call() 方式**: `IndexManager` のコンポーネントIDを `Folder/Tree` に Prop Drill し、フロントエンドから `Livewire.find(parentId).call('changeCurrentFolder', id)` を実行。これにより、親コンポーネントの `wire:target` が正しく発火し、スケルトン表示とコンテンツ更新の完全同期に成功。

**現在のステータス**: 
- **解決済み**: ドロワー経由の操作でも、メインエリアのスケルトン切り替えが即座に行われるようになり、UX上の違和感が解消された。

### 2. [完了] スピナーの位置ずれと中央固定
**現象**:
- 全画面ローディングオーバーレイ使用時、スピナーが画面上部に張り付いたり、コンテンツの高さに引きずられて位置が定まらない。

**解決策**:
- `IndexManager` のグローバルローダーを `fixed inset-0 flex items-center justify-center` なコンテナでラップ。
- 子コンポーネント側に `manual` プロップを渡し、位置制御を親に一任することで、ビューポート中央への物理的な固定を実現。

### 3. [完了] ローディング表示タイミング（煩わしさ）の改善
**現象**:
- `.delay.long` (約300ms) では、高速なフォルダ移動の際にもスピナーが一瞬見えてしまい、ユーザーにストレスを与える。

**解決策**:
- グローバルスピナーの遅延を `.delay.longest` (1000ms) に設定。
- これにより「1秒以内の通常操作はスケルトンのみ」「1秒を超える重い処理はスピナー」という疎密のあるフィードバックを実現。

### 4. [完了] Reactive Prop Mutation Error
**現象**:
- `RecordsTable` で親から渡された `displayLevel` を直接 `wire:model` バインドしたことで、変更時に親プロパティを破壊しようとしてエラーが発生。

**解決策**:
- Alpine.js の `$watch` を使い、入力値を検知してから親のメソッド (`updateDisplayLevel`) を明示的に叩く方式へ変更。Livewire 3 の Reactive 原則（子は親プロパティを変更できない）を遵守した実装へ修正。
