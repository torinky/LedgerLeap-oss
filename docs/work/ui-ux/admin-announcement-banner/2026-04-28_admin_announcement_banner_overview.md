# 管理者お知らせ機能の検討メモ

- date: 2026-04-28
- scope: 上部強制表示を主役にした管理者向けお知らせ
- goal: 別機能として見える登録管理 UI を持たせつつ、通知センターへ裏側統合する

## 1. まずの結論

既存の通知センターと database 通知基盤がすでにあるため、裏側の統合先としては活用できます。
ただし主UIはページ上部の強制表示に置き、別機能としての存在感を最初から持たせるべきです。

## 2. 背景と狙い

LedgerLeap には通知一覧、未読管理、ワークフロー通知、グローバルな通知ページがすでにあります。
運用上は、上部表示で着意させたうえで、通知センターを再確認や履歴の受け皿として使うのが自然です。

## 3. 想定ペルソナ

### 3.1 管理者

- システム全体の案内を出したい
- メンテナンス予告や障害連絡を予約したい
- 表示期間と優先度を指定したい
- 既に終了した案内を自動で消したい
- 登録・公開・停止を管理画面で分けて扱いたい

### 3.2 一般ユーザー

- 作業の邪魔にならない範囲で、重要な運用情報だけ受け取りたい
- 緊急度が高い案内は見逃したくない
- 詳細が必要な場合は、そのまま関連ページへ移動したい

### 3.3 運用担当

- いつからいつまで出すかを事前に決めたい
- 過去に何を出したかを追跡したい
- どの表示レベルがどのケースに対応するかを統一したい

## 4. ユースケース

### 4.1 定期メンテナンスの予告

- 数日前から告知したい
- 開始時刻と終了時刻を予約したい
- 終了後は自動的に非表示にしたい

### 4.2 障害・遅延のお知らせ

- 全体影響のある障害を強調したい
- 必要なら復旧案内や障害詳細へのリンクを出したい
- 重要度に応じて最上位表示にしたい

### 4.3 運用ルール変更

- 操作や承認のルール変更を周知したい
- すぐには対応不要でも、一定期間は常時見えるようにしたい

### 4.4 締切・期限の注意喚起

- 期限が近い対象者だけに出したい
- 対象者ごとに表示するか、全体告知にするかを切り替えたい

## 5. 画面要件

### 5.1 表示位置

- ページ上部にまず見せる
- 通知センターでは同じ告知を再確認できる
- 固定ヘッダーやタブ列と重なっても、着意が先に立つ設計を優先する

### 5.2 見た目

- 半透明のステッカーのような軽さを持たせる
- ただし装飾を優先しすぎず、可読性を保つ
- 強い警告は色と文言で区別し、色だけに頼らない

### 5.3 ふるまい

- 表示開始日時と表示終了日時を持つ
- 予約公開と自動停止ができる
- 優先度で並べ替えできる
- 複数件を同時に出すか、最上位1件だけを出すかを選べる

### 5.4 操作

- 登録、公開、停止を分けた管理操作を持つ
- 閉じるボタンを付けるかどうかをレベル別に制御する
- 詳細ページへのリンクや対応ボタンを持てる
- 閉じた後の再表示条件を決められるようにする

## 6. 仕様案

### 6.1 表示レベル

最低でも次の4段階があると運用しやすいです。

- info: 案内・周知
- success: 完了報告・反映済み
- warning: 注意・期限接近・条件付きの警告
- critical: 障害・停止・即時対応が必要な告知

### 6.2 表示ルール

- critical は常に最上位で見えることを優先する
- warning は注意喚起として目立たせるが、作業を完全に止めない
- info は控えめに表示し、邪魔になりすぎない
- success は短命にするか、管理者向けの操作完了表示に限定する

### 6.3 開始・停止

- starts_at と ends_at を持つ
- タイムゾーンはシステム基準を明示する
- 予約時刻未到来なら非表示、終了後は自動失効とする

### 6.4 対象範囲

- 全体共通
- テナント単位
- 役割単位
- 特定ユーザー単位

この4種を切り替えられると、運用の幅が一気に広がります。

### 6.5 再表示と既読

- 閉じたら二度と出さないのか
- セッションごとに再表示するのか
- ユーザー既読を保持するのか

この3つは別物として扱うべきです。

## 7. 未決めの論点

- 上部強制表示を全画面共通にするか、特定範囲に限定するか
- テナント共通とシステム全体を分けるか
- 1件だけ表示するか、複数件を積むか
- 既読保存をユーザー単位にするか
- critical を必ず閉じ不可にするか
- 管理者プレビュー画面を別途用意するか
- 通知センターは履歴だけでなく、再確認導線としてどう見せるか

## 8. 次の進め方

1. 上部強制表示を主UIとして先に固める
2. 登録管理 UI と公開表示 UI の責務を分ける
3. 通知センター側の統合表示を補助経路として定義する
4. 表示ルールを状態遷移図にする
5. 画面モックまたは Livewire の試作に進む
