# Ledger IndexManager UI 見直し計画

**作成日**: 2026-04-11  
**対象**: `resources/views/livewire/ledger/index-manager.blade.php` / `resources/views/components/ledger/search.blade.php` / 関連 Livewire テスト

## 目的

台帳一覧画面の「検索・ツリー操作・複数台帳表示・ローディング」の関係を整理し、
ユーザーが操作中に迷いにくい情報配置へ再設計する。
特に、表示レベル調整を検索オプション側へ寄せる案を中心に、
検索結果件数・ページング・表示レベルの配置をユーザーシナリオに合わせて見直す。

## 背景

- 検索やツリー操作により、複数の台帳が同時に表示されることがある。
- 操作に応じてローディング表現が切り替わるため、表示崩れやチラつきを避ける必要がある。
- 現在は検索ヘッダー、情報ブロック、結果領域で情報の役割がやや分散している。
- 表示レベルは「表示密度の調整」であり、検索条件の一種として扱う方が自然な可能性がある。

## 見直し方針

### 1. 表示レベルの配置を検索オプションへ寄せる
- `displayLevel` を検索オプションブロックの一部として再配置する。
- 表示密度の調整を、検索・絞り込みと同じ文脈で操作できるようにする。
- 現在の情報ブロックにある表示レベル UI は整理または縮小する。

### 2. 検索結果件数の役割を明確化する
- 件数表示は「現在の検索条件で何件ヒットしているか」を示す情報に寄せる。
- 複数台帳が表示される状況でも、件数表示が状態把握に集中するようにする。
- ローディング中はプレースホルダー表示を維持し、表示切替で高さが不安定にならないようにする。

### 3. ページングの配置をユーザー導線に合わせる
- ページングは検索オプションには寄せず、結果領域側の導線に残す。
- 使い方として「検索条件の調整」と「一覧をめくる操作」を分けて理解しやすくする。
- 複数台帳表示でもページ位置を見失わない配置を優先し、一覧の読み進めと検索条件の変更を混同させない。

### 4. ローディング表現を壊さない
- `wire:loading` の対象は維持し、重い更新と軽い更新の扱いを変えない。
- ツリー操作・検索・表示レベル変更で、スケルトン/透過/スピナーの境界を壊さない。
- 複数台帳が同時に見えるケースでも、表示領域の高さやカード配置が急変しないようにする。

### 5. 文言を自然な日本語へ調整する
- 検索、表示レベル、件数、ページングのラベルを役割に合わせて短く整理する。
- 既存の翻訳キーを優先し、必要であれば `lang/ja/ledger/` 側を更新する。
- ハードコードされた自然言語は増やさない。

## テスト観点

1. 検索語を変えたときに表示レベルや件数表示が意図どおり更新されること。
2. フォルダ切り替え時にローディング表現が崩れないこと。
3. 複数台帳が表示される状態でも、情報ブロックの高さと配置が破綻しないこと。
4. 表示レベルの変更が検索オプション側へ移動しても、状態更新が壊れないこと。
5. 既存の `wire:target` / `wire:key` / `x-show` 挙動が維持されること。

## 完了条件

- 検索・表示レベル・件数・ページングの配置がユーザーの操作順に沿って整理されている。
- ローディング中の表示が維持され、視覚的な破綻がない。
- 関連テストが追加または更新され、変更後の回帰を防げる。

## 実施後の差分メモ

このタスクでの最終結果は、当初の「検索ヘッダーに情報を寄せる」発想から少し分離した。

- `displayLevel` は検索オプションブロックへ移動し、密度調整として扱う。
- 検索結果件数は、常時表示の情報ブロック側で状態把握のために残す。
- ページングは検索ヘッダーへ入れず、`records-table` 側の結果導線に残す。
- 複数台帳が同時に表示される前提では、軽い更新時に DOM を壊さず、loading は透過/overlay で吸収する。
- 再調整時は「検索条件の調整」「結果の把握」「ページをめくる操作」を同じ塊にしないことを優先する。
