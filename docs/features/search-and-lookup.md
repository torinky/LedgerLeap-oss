# 検索とルックアップ

## 概要

LedgerLeap の検索機能は、台帳の内容や添付ファイルを対象に、キーワード検索、タグ検索、条件絞り込み、類義語展開、関連案件の自動発見など、複数の方法で目的の情報を素早く見つけられるようにする仕組みです。

## 検索が必要な理由

台帳の件数が増えると、フォルダをたどって目的のレコードを探すのに時間がかかります。検索機能を使うと、次のような課題を解決できます。

- **キーワードが不明確でも見つけられる**: 類義語や技術用語の展開により、正確な用語を知らなくても目的のレコードにたどり着ける。
- **複数の条件を組み合わせて絞り込める**: フォルダ、タグ、作成者、日付範囲、台帳定義を組み合わせて検索できる。
- **関連するレコードを横断的に発見できる**: 識別番号や意味的な類似性に基づいて、他の台帳定義に属する関連レコードを自動的に表示できる。

## 検索の種類と方法

### キーワード検索

画面上部の検索ボックスにキーワードを入力すると、テナント内の台帳を横断検索します。検索対象は次の項目です。

- 台帳の全カラムの値（テキスト、数値、日付、Markdown など）
- 添付ファイルのファイル名
- OCR で抽出されたテキスト（対応ファイル形式の場合）

検索は Mroonga フルテキスト検索エンジンを使用しており、日本語の形態素解析に対応しています。

### 類義語検索

管理者が類義語を登録しておくと、検索キーワードが自動的に展開されます。たとえば、「点検」というキーワードで検索したときに、「検査」「確認」「チェック」を類義語として登録しておけば、これらの語を含むレコードも同時に検索されます。

類義語の管理は管理者画面（Filament）の類義語リソースから行います。

### 技術用語検索

技術用語や社内コードなど、特定の用語を「技術用語」として登録しておくと、検索時に自動的に考慮されます。技術用語は類義語とは異なり、略称と正式名称の対応づけなどに使用します。

### タグ検索

台帳に付与されたタグを使って検索できます。タグはテナントごとに管理され、複数のタグを組み合わせた検索も可能です。タグの管理は管理者画面から行います。

### 条件絞り込み検索

キーワード検索に加えて、次の条件で絞り込めます。

- **フォルダ**: 特定のフォルダとその子フォルダに範囲を限定する。
- **台帳定義**: 特定の台帳定義（テンプレート）に限定する。
- **作成者**: 特定のユーザーが作成したレコードに限定する。
- **作成日範囲**: 指定した期間内に作成されたレコードに限定する。
- **ステータス**: ワークフローのステータスで絞り込む。
- **添付ファイルの有無**: 添付ファイルがあるレコードのみに限定する。

### ルックアップ（自動リンク）

台帳の詳細画面で、他のレコードを参照するための自動リンク機能があります。識別番号（自動採番カラムの値）がテキスト中に記載されている場合、該当部分が自動的にリンクになり、クリックすると参照先の台帳詳細画面に移動します。

### 関連案件

台帳の詳細画面の「関連案件」タブでは、閲覧中のレコードに関連する他のレコードを次の 2 つの方法で横断検索します。

- **識別番号検索**: レコードが持つ識別番号を使って、テナント内の全台帳を横断検索する。
- **意味検索（RAG）**: レコードの内容をクエリとして、コサイン類似度に基づく意味的な検索を実行する。

関連案件の結果は、識別番号一致、意味検索一致、またはその両方として表示され、スコア順に並びます。

## 検索方法の使い分け

| 方法 | いつ使うか |
|---|---|
| キーワード検索 | 検索ボックスに思いついた単語を入れてざっと探す |
| 類義語検索 | 同じ意味の別の言葉でもヒットさせたい |
| タグ検索 | 特定の分類に属するレコードをまとめて見つける |
| 条件絞り込み | フォルダや日付などで範囲を限定して探す |
| ルックアップ | 台帳の記載内容から関連レコードへジャンプする |
| 関連案件 | 現在見ているレコードに関連するものを自動的に探す |

## API による検索

検索機能は REST API からも利用できます。`GET /api/v1/search` または `POST /api/v1/search` エンドポイントを使って、プログラムから検索を実行できます。API の詳細は Search API リファレンスを参照してください。

## 注意点と制約

- 検索は現在のテナント内に限定されます。別のテナントのデータは検索結果に含まれません。
- 全文検索は Mroonga（単一カラムの MATCH AGAINST）を使用しています。複合インデックスは使用できません。
- 類義語や技術用語の展開は、管理者が事前に登録しておく必要があります。
- 意味検索（RAG）は RAG サービスが有効な環境でのみ利用できます。サービスが未起動の場合は識別番号検索のみが有効です。
