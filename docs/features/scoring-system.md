# スコアリングシステム

**バージョン:** 1.0.0  
**最終更新:** 2025年10月13日  
**ステータス:** ✅ Phase 1・1.5 実装完了

---

## 概要

LedgerLeapのスコアリングシステムは、台帳データに対して複数の評価指標を組み合わせた「ハイブリッド型情報価値評価」を行い、ユーザーが重要な情報を迅速に発見できるようにする機能です。

### 主要機能

1. **複合スコア計算** - 活動スコア、新鮮度スコア、重要度スコアを統合
2. **自動ソート** - スコアに基づいて台帳を自動的に優先順位付け
3. **視覚的表示** - スコアバッジによる直感的な優先度表示
4. **環境別頻度制御** - 開発環境と本番環境で異なる更新頻度

---

## スコア計算式

### 複合スコア（Composite Score）

```
複合スコア = (活動スコア × 40%) + (新鮮度スコア × 30%) + (重要度スコア × 30%)
```

各スコアは 0〜100 の範囲で正規化されます。

### 1. 活動スコア（Activity Score）

直近の操作頻度を反映します。

**計算方法:**
```
活動スコア = (直近7日間のイベント数 × 10) + (直近30日間のイベント数 × 3)
```

**対象イベント:**
- `created` - 作成
- `updated` - 更新
- `commented` - コメント追加
- `viewed` - 閲覧
- `downloaded` - ダウンロード

**スコア例:**
- 直近7日間に3回更新 → 30点
- 直近30日間に5回閲覧 → 15点
- 合計: 45点

### 2. 新鮮度スコア（Freshness Score）

更新日時からの経過時間を反映します。

**計算方法:**
```php
$daysSinceUpdate = now()->diffInDays($ledger->updated_at);
$freshnessScore = max(0, 100 - ($daysSinceUpdate * 2));
```

**スコア例:**
- 今日更新: 100点
- 10日前: 80点
- 30日前: 40点
- 50日以上前: 0点

### 3. 重要度スコア（Importance Score）

ワークフローステータスに基づく重要度を反映します。

**スコア定義:**
- `承認待ち (PENDING)`: 100点
- `差し戻し (RETURNED)`: 80点
- `検査中 (IN_REVIEW)`: 60点
- `下書き (DRAFT)`: 20点
- `承認済み (APPROVED)`: 10点
- その他: 0点

## 使用方法

### スコアの表示

台帳一覧画面では、各レコードにスコアバッジが表示されます：

- **🟢 緑 (70〜100点)**: 高優先度
- **🔵 青 (40〜69点)**: 中優先度
- **🔷 水色 (20〜39点)**: 低優先度
- **⚪ グレー (1〜19点)**: 最低優先度

### デフォルトソート

台帳一覧は自動的に複合スコアの降順（高い順）でソートされます。従来の作成日順や更新日順でのソートも選択可能です。

### ソート切り替え

統合ツールバーの「並び順」ドロップダウンから選択：

- **スコア順** - 複合スコア降順（デフォルト）
- **作成日順** - 作成日の新しい順
- **更新日順** - 更新日の新しい順
- **カスタムカラム** - 台帳定義のカラムで直接ソート可能

### 台帳定義別スコア統計

検索時は、台帳定義ヘッダーにスコア統計が表示されます：

- **平均スコア** - その台帳定義の平均複合スコア
- **最高スコア** - 最も高い複合スコア
- **レコード数** - 該当レコード数

台帳定義は平均スコアの高い順に表示されます。

## スコア更新頻度

スコアは定期的なバッチ処理で自動更新されます。

### 環境別の推奨設定

| 環境 | 頻度 | 設定値 | 用途 |
|------|------|--------|------|
| **開発・デモ** | 5分ごと | `everyFiveMinutes` | リアルタイム確認 |
| **本番（アクティブ）** | 毎時 | `hourly` | 活発な環境 |
| **本番（標準）** | 毎日 | `daily` | 通常運用 |
| **本番（大規模）** | 毎週 | `weekly` | 負荷軽減 |

### 設定方法

`.env` ファイルで設定：

```bash
# 開発環境
SCORING_SCHEDULE_FREQUENCY=everyFiveMinutes

# 本番環境
SCORING_SCHEDULE_FREQUENCY=daily
```

設定変更後は scheduler コンテナを再起動：

```bash
./vendor/bin/sail restart scheduler
```

## 技術仕様

### データベーステーブル

#### ledgers テーブル（追加カラム）

| カラム名 | 型 | 説明 |
|---------|----|----|
| `activity_score` | DECIMAL(5,2) | 活動スコア（0-100） |
| `composite_score` | DECIMAL(5,2) | 複合スコア（0-100） |

インデックス:
- `idx_ledgers_composite_score` - 複合スコアのインデックス

### 設定ファイル

#### config/ledgerleap.php

```php
'scoring' => [
    'activity' => [
        'windows' => [
            ['days' => 7, 'multiplier' => 10],
            ['days' => 30, 'multiplier' => 3],
        ],
    ],
    'weights' => [
        'activity' => 0.40,
        'freshness' => 0.30,
        'importance' => 0.30,
    ],
    'schedule_frequency' => env('SCORING_SCHEDULE_FREQUENCY', 'daily'),
],
```

### Artisanコマンド

#### スコア計算コマンド

```bash
php artisan scoring:calculate
```

全ての台帳のスコアを再計算します。

**オプション:**
- 進捗バー表示
- テナント別処理
- エラーログ記録

**実行時間:** 1万レコードあたり約2〜3分

## トラブルシューティング

### スコアが更新されない

**原因:** scheduler コンテナが停止している

**解決策:**
```bash
./vendor/bin/sail restart scheduler
```

### スコアが 0 のまま

**原因:** バッチ処理がまだ実行されていない

**解決策:**
```bash
# 手動で実行
./vendor/bin/sail artisan scoring:calculate
```

### 開発環境でスコアがすぐに更新されない

**原因:** 更新頻度が daily になっている

**解決策:**
```bash
# .env に追加
SCORING_SCHEDULE_FREQUENCY=everyFiveMinutes

# scheduler 再起動
./vendor/bin/sail restart scheduler
```
