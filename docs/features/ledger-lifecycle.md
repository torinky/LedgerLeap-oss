# 台帳ライフサイクル

## 概要

台帳（Ledger）は、LedgerLeap におけるデータの基本単位です。台帳定義（LedgerDefine）で定義されたカラム構造に従って作成され、作成から参照、更新、複製、エクスポートまでの一連の流れ（ライフサイクル）を持ちます。このページでは、台帳のライフサイクルと各操作の概要を説明します。

## 台帳のライフサイクルとは

台帳は、次のような段階を経て管理されます。

1. **台帳定義の選択** — どのテンプレート（台帳定義）を使ってデータを管理するか決める
2. **作成** — 台帳定義に基づいて新しいレコードを作成する
3. **参照** — 一覧または詳細画面でデータを確認する
4. **更新** — 既存のレコードの内容を編集する
5. **複製** — 既存のレコードをコピーして新しいレコードを作成する
6. **インポート・エクスポート** — Excel や CSV を使って一括でデータを入出力する
7. **削除** — 不要になったレコードを削除する
8. **変更履歴の確認** — 各更新の差分を追跡する

ワークフローが有効な台帳定義では、作成・更新のたびに点検や承認のプロセスが挟まります。ワークフローの詳細は別ページで説明します。

## ライフサイクルの各段階

### 台帳一覧の表示

フォルダツリーから目的のフォルダを選ぶと、そのフォルダに属する台帳の一覧が表示されます。一覧では、各レコードの要約やステータス、スコア、添付ファイルの有無などを確認できます。

一覧の並び順は、複合スコア（活動頻度、更新時期、重要度を組み合わせた優先度）がデフォルトです。作成日順や更新日順に変更することもできます。

### 台帳の作成

台帳一覧画面またはフォルダ画面から「新規作成」を選択すると、台帳定義の選択画面に移動します。利用したい台帳定義を選び、各カラムに値を入力して保存します。

作成時には次の情報を入力できます。

- 台帳定義で定義された各カラムの値（テキスト、数値、日付、Markdown、自動採番など）
- 添付ファイル
- タグ

ワークフローが有効な台帳定義の場合、作成直後のステータスは「下書き（DRAFT）」になります。

### 台帳の詳細表示

台帳一覧でレコード名をクリックすると、詳細画面が開きます。詳細画面では次の情報を確認できます。

- 各カラムの値
- 添付ファイルの一覧とプレビュー
- 変更履歴（差分ビューア）
- 関連案件（識別番号や意味的な類似性に基づく関連レコード）
- アクティビティログ
- 権限情報

### 台帳の編集

詳細画面から「編集」ボタンをクリックすると、編集画面に移動します。変更したいカラムの値を修正して保存します。

編集時の動作はワークフローの設定に依存します。

- ワークフローが無効: 保存すると即座に内容が更新されます。
- ワークフローが有効: 保存するとステータスが「下書き」に戻り、再度点検・承認プロセスが必要になります。

### 台帳の複製

既存のレコードをコピーして新しいレコードを作成できます。複製時には、元のレコードのカラム値が引き継がれた状態で作成画面が開くため、似た内容のレコードを効率よく作成できます。

### インポート

Excel または CSV ファイルを使って、複数のレコードを一括で登録できます。インポート機能では次の操作が可能です。

- 新規レコードの追加
- 既存レコードの更新（識別番号をキーにマッチング）
- インポート前のプレビュー

### エクスポート

台帳一覧の内容を Excel または CSV 形式でダウンロードできます。現在の一覧に適用されているフィルターやソート順がエクスポート結果にも反映されます。

### 削除

不要になったレコードは削除できます。削除したレコードは元に戻せません。

### 変更履歴の確認

台帳が更新されるたびに、変更前後の差分が自動的に記録されます。詳細画面の履歴タブでは次の情報を確認できます。

- 更新日時と更新者
- 変更前後のカラムごとの差分
- 更新時に添付されたコメント

## 各操作の使い分け

| 操作 | いつ使うか |
|---|---|
| 新規作成 | 新しいデータを 1 件ずつ登録する |
| 編集 | 既存のデータを修正する |
| 複製 | 既存のデータをベースに新しいデータを作る |
| インポート | 外部データを一括で取り込む |
| エクスポート | データを外部で利用するために書き出す |
| 削除 | 不要なデータを完全に除去する |

## 注意点と制約

- 台帳の作成や編集には、フォルダに対する適切な権限が必要です。
- ワークフローが有効な台帳定義では、承認後の編集は原則としてできません。編集が必要な場合は、差し戻しまたはロールバックの手続きが必要です。
- インポートは大量データの一括処理に向いていますが、インポート中は画面を閉じずに待つ必要があります。
- 削除したレコードは復元できません。削除前に内容をよく確認してください。
- 変更履歴は自動的に記録されますが、保持期間に上限がある場合があります。
