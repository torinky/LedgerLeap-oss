LedgerLeapシステムのMCPサーバーを効率的に活用するためのプロンプトガイドラインです。

> **📖 技術的な背景:** MCPプロトコルの詳細な動作メカニズムや`instructions`プロパティの技術的な役割については、[MCP アーキテクチャと動作フロー](./MCP_Architecture_and_Flow.md)を参照してください。

## 基本指針

### 1. 応答形式の最適化

**`__summary__` フィールドの活用:**
- MCPツールからの応答に `__summary__` が含まれている場合、それを応答の冒頭で使用してください
- 例: "あなたが作成した台帳は3件です。" → そのまま使用

**`__display_fields__` の活用:**
- リスト形式のデータには `__display_fields__` が含まれています
- 日本語のフィールド名（件名、ステータス、更新日時）を使用して見やすく整理してください

### 2. 検索クエリの処理パターン

#### パターン1: 時間ベースの検索
**ユーザー入力例:**
- "昨日私が作成した日報を見せて"
- "今週作成された台帳を確認したい"
- "2025年1月18日の記録を探して"

**推奨ツール呼び出し:**
```json
{
  "tool": "SearchLedgers",
  "arguments": {
    "creator_id": "[現在のユーザーID]",
    "created_from": "2025-01-18",
    "created_to": "2025-01-18", 
    "format": "summary",
    "q": "日報"
  }
}
```

#### パターン2: キーワード検索
**ユーザー入力例:**
- "営業に関する台帳を探して"
- "テストという言葉が入った記録"

**推奨ツール呼び出し:**
```json
{
  "tool": "SearchLedgers", 
  "arguments": {
    "q": "営業",
    "format": "summary",
    "limit": 10
  }
}
```

#### パターン3: 作成者フィルタ
**ユーザー入力例:**
- "私の台帳一覧"
- "自分が作った記録"

**推奨ツール呼び出し:**
```json
{
  "tool": "SearchLedgers",
  "arguments": {
    "creator_id": "[現在のユーザーID]",
    "format": "summary"
  }
}
```

### 3. 応答テンプレート

#### 検索結果がある場合
```
[__summary__の内容]

📋 **見つかった台帳:**
• **件名:** [台帳タイトル]
  - **ステータス:** [ステータス]
  - **更新日時:** [日時]

• **件名:** [台帳タイトル] 
  - **ステータス:** [ステータス]
  - **更新日時:** [日時]

何か他にお探しのものはありますか？
```

#### 検索結果がない場合
```
申し訳ございませんが、指定された条件に該当する台帳は見つかりませんでした。

以下をご確認ください：
- 検索キーワードを変更してみる
- 日付範囲を広げてみる
- フィルタ条件を緩くしてみる

他の検索条件でお試しいただくか、新しい台帳を作成されますか？
```

### 4. エラーハンドリング

**認証エラーの場合:**
```
申し訳ございませんが、認証に問題があります。
ログインし直すか、システム管理者にご相談ください。
```

**権限エラーの場合:**
```
この操作を実行する権限がありません。
必要な権限については、システム管理者にお問い合わせください。
```

### 5. 台帳作成の支援

**台帳定義の確認:**
```json
{
  "tool": "GetLedgerDefines"
}
```

**台帳作成:**
```json
{
  "tool": "CreateLedger",
  "arguments": {
    "ledger_define_id": "[選択された定義ID]",
    "folder_id": "[適切なフォルダID]", 
    "content": "[ユーザー入力に基づく内容]"
  }
}
```

## 実装例

### ユーザー: "昨日私が作成した日報を見せて"

**1. ツール呼び出し:**
```json
{
  "tool": "SearchLedgers",
  "arguments": {
    "creator_id": 1,
    "created_from": "2025-01-18",
    "created_to": "2025-01-18",
    "q": "日報", 
    "format": "summary"
  }
}
```

**2. LLM応答:**
```
あなたが昨日作成した台帳は2件です。

📋 **見つかった日報:**
• **件名:** 2025年1月18日営業日報
  - **ステータス:** 承認待ち
  - **更新日時:** 2025年1月18日 18:30

• **件名:** 2025年1月18日作業報告
  - **ステータス:** 下書き  
  - **更新日時:** 2025年1月18日 20:15

他にも何かお手伝いできることはありますか？
```

## 注意事項

1. **日付の処理**: `created_from` と `created_to` は YYYY-MM-DD 形式で指定
2. **creator_id**: ユーザーが「私の」「自分の」と言った場合は現在のユーザーIDを使用
3. **format パラメータ**: 必ず `"summary"` を使用して見やすい応答を取得
4. **エラー処理**: ツールからエラーが返された場合は、ユーザーにとって理解しやすい形で説明
