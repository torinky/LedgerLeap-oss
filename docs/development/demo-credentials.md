| フォルダ | Super Admin | 一般（営業） | 一般（技術） | 点検者 | 承認者 |

| / (ルート) | ADMIN | - | - | - | - |
| 営業部/* | 継承 | WRITE | - | - | - |
| 技術部/* | 継承 | - | WRITE | - | - |
| 全社共通/* | 継承 | WRITE | WRITE | - | - |

### 権限レベルの説明

| 権限レベル | 説明 |
|-----------|------|
| **ADMIN** | 全ての操作が可能（作成・読取・更新・削除・権限管理） |
| **WRITE** | 作成・読取・更新が可能 |
| **READ** | 読取のみ可能 |
| **継承** | 親フォルダの権限を継承 |

#### 2. 営業ユーザーでログイン
```
メール: sales1@example.com

```

**確認項目:**
- [ ] 営業部フォルダが表示される
- [ ] 全社共通フォルダが表示される
- [ ] 技術部フォルダは表示されない（権限なし）
- [ ] 営業日報の作成・編集が可能

#### 3. 開発ユーザーでログイン
```
メール: dev1@example.com
パスワード: demo1234
```

**確認項目:**
- [ ] 技術部フォルダが表示される
- [ ] 全社共通フォルダが表示される
- [ ] 営業部フォルダは表示されない（権限なし）
- [ ] 週報の作成・編集が可能

### ワークフローテスト

#### 経費申請フロー

1. **申請者（営業太郎）でログイン**
   - 経費申請を新規作成
   - ステータス: DRAFT（下書き）

2. **点検者（点検一郎）でログイン**
   - 経費申請を点検
   - ステータス: PENDING_INSPECTION → PENDING_APPROVAL

3. **承認者（承認一郎）でログイン**
   - 経費申請を承認
   - ステータス: PENDING_APPROVAL → APPROVED
