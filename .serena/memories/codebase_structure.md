
# LedgerLeap Codebase Structure

This project follows a standard Laravel application structure with additional directories for specific functionalities like Livewire components, Filament admin panels, and custom services.

-   `app/`: Contains the core application logic.
    -   `Casts/`: Custom Eloquent casts.
    -   `Console/`: Artisan commands.
    -   `Database/`: Database migrations, seeders, and factories.
    -   `Enums/`: PHP Enums for various application states and types.
    -   `Exceptions/`: Custom exception classes.
    -   `Exports/`: Classes for exporting data (e.g., Excel).
    -   `Facades/`: Custom Facades.
    -   `Filament/`: Filament admin panel resources, pages, and widgets.
    -   `Forms/`: Form-related classes.
    -   `Helpers/`: Helper functions.
    -   `Http/`: HTTP layer (Controllers, Middleware, Requests).
    -   `Imports/`: Classes for importing data.
    -   `Jobs/`: Queueable jobs.
    -   `Livewire/`: Livewire components.
    -   `Mail/`: Mailable classes.
    -   `Models/`: Eloquent models.
    -   `Modules/`: Potentially for modularizing larger features.
    -   `Notifications/`: Notification classes.
    -   `Observers/`: Eloquent model observers.
    -   `Policies/`: Authorization policies.
    -   `Providers/`: Service providers.
    -   `Repositories/`: Repository classes for data access.
    -   `Rules/`: Custom validation rules.
    -   `Services/`: Business logic services (fat controllers are avoided).
    -   `Traits/`: Reusable PHP traits.
    -   `View/`: View-related classes (e.g., Composers).
-   `bootstrap/`: Framework bootstrap script and cache directory.
-   `config/`: Application configuration files.
-   `database/`: Database migrations, factories, and seeders.
-   `docker/`: Docker-related configurations for development and production environments.
-   `docs/`: Project documentation.
-   `lang/`: Language files for internationalization.
-   `public/`: Web server entry point and assets.
-   `resources/`: Frontend assets (JavaScript, CSS, SASS) and Blade templates.
-   `routes/`: Web, API, console, and channel routes.
-   `scripts/`: Custom shell scripts (e.g., `dev.sh`, `prod.sh`).
-   `storage/`: Compiled Blade templates, file sessions, file caches, and other files generated by the framework.
-   `tests/`: Automated tests (Unit and Feature).
    -   `Feature/`: Feature tests.
    -   `Unit/`: Unit tests.
-   `vendor/`: Composer dependencies.
